# Phase 7 — UI Design Contract: Graph-Based Codebase Map

> Visual and interaction contract for the force-directed graph radar that replaces the Phase 4 squarified-treemap radar. Generated by gsd-ui-researcher against locked decisions D-01..D-24 in `07-CONTEXT.md`. Verified by gsd-ui-checker downstream.

Scope: this phase is a **replacement** of the `RadarCanvas` layout/render body. It inherits the Phase 4 UI-SPEC for everything that is preserved unchanged (manifest panel, heat map toggle, notification copy, diff/chat surfaces, StatusBadge, RadarPulse, zero-radius rule, fonts, surface hierarchy). Only deltas and new surfaces are specified here.

### Canvas-rendered text sizing

Canvas 2D `font` strings in pixel units. For world-space text, divide by `viewport.zoom` so label reads at the same visual size regardless of zoom:

```js
ctx.font = `${12 / zoom}px "Space Grotesk", sans-serif`;     // top-level folder label
ctx.font = `${10 / zoom}px "Space Grotesk", sans-serif`;     // nested folder label
```

Exception: DOM-overlay text (tooltip, minimap, banner) uses fixed CSS pixel sizes from the table above — do not scale by viewport zoom.

### Typography rules (inherited + phase-specific)

- Space Grotesk is always UPPERCASE with wide tracking for labels (Phase 1 rule).
- JetBrains Mono used for all data: file paths, agent IDs, numeric counts.
- Never use pure white `#ffffff` for body text; use `on-surface-variant` `#adaaaa`. Reserve `#ffffff` for selected-node highlight text and critical headers only.
- Folder labels render at **60% opacity** for top-level, **40% opacity** for nested — so labels recede behind the node/edge foreground (D-12 "outlines and labels recede").
- Tooltip typography matches Phase 4 exactly; not redefined here.

### Heat-map color ramp (D-19, replaces treemap rect tint)

Node fill blend function (score ∈ [0, 1]):

```
heatColor(score) = mixRgb(
  surface-container #1a1919,
  error            #ff7351,
  clamp(score, 0, 1)
)
```

At score = 1.0, a node is rendered fully in `#ff7351`. Reuse `computeContentionScore` from `src/lib/contention.ts` **unchanged** — only the render path changes (node fill instead of rect fill). No new data-layer code.

### The No-Line Rule (inherited)

Borders remain prohibited for sectioning **around HTML surfaces**. Phase 7 boundaries defined through:
1. Tonal shifts (manifest panel `surface-container-low` vs radar canvas `surface-container-lowest`).
2. Negative space (12px gutter around minimap and toggle).
3. Ghost borders (outline-variant at 10–15% opacity) permitted only for: AgentTooltip container, minimap container, performance warning banner.

**Note:** Graph edges, arrow heads, folder hulls, and node strokes are rendered on the Canvas itself and are signal — not UI chrome — so the no-line rule does not apply to them.

### Progressive Detail Tiers (D-12, D-23)

| Zoom | Render |
|------|--------|
| < 0.6× | Only top-level folder hulls + labels. Node dots rendered but without strokes. Edges rendered at 30% opacity. Arrow heads culled. |
| 0.6× – 2× | Depth ≤ 2 folder hulls + labels. Nodes at full fidelity. Edges + arrow heads full opacity. |
| ≥ 2× | All folder hulls + labels (all depths). Nodes, edges, arrow heads full fidelity. File-name tooltip available on hover. |
| ≥ 4× | File-name tooltip shown automatically for nodes under cursor (matches Phase 4 zoom-3x file labels cadence). |

Viewport culling: nodes and edges whose bounding boxes are entirely outside the current viewport (plus a 100px padding margin) are skipped in the render loop. Critical for D-23 10k-node target.

### Render z-order (Canvas 2D draw order, bottom-to-top)

The render loop must draw layers in this exact order so that agent trails and pulses sit above the spatial graph:

1. Canvas clear (surface-container-lowest, #000000)
2. Folder hulls — fill then stroke
3. Folder labels (text on hull centroid)
4. Edges — stroke only
5. Arrow heads — fill
6. Graph nodes — fill (heat-tinted when toggle on) then stroke
7. Selected-node outer halo (1px white stroke at 80%)
8. Selected-agent ambient glow (radial gradient)
9. Comet tails — per-segment gradient strokes (oldest first, newest on top)
10. Comet heads — fill + outer glow
11. Agent current-position dots — pulse rings (outer, inner), then center dot
12. Conflict pulse rings (expanding concentric rings)
13. Conflict badge dots + pinned-node lock badges (always-on markers)

Any labels that overlap nodes should be drawn **before** nodes (step 3) so nodes sit on top of labels, preserving node hit-testability. Top-level folder labels may render above nodes at low zoom (< 0.6×) — at that zoom nodes are dense dots and label readability wins.

## Motion

All timings from either the Phase 4 inherited animations, Command Horizon "Phosphor Transition" spec, or research-driven recommendations. Every animation must honor `prefers-reduced-motion` (see Accessibility).

| Animation | Spec | Usage |
|-----------|------|-------|
| Phosphor Transition (inherited) | 150ms opacity fade + subtle vertical scanline wipe | Empty-state appearance, manifest toggle content crossfade |
| **Graph settle → visible** (D-03) | 200ms opacity fade-in (cubic ease-out) once `settledAt` is set | One-shot on initial settle. Do NOT animate nodes from origin outward — they appear pre-settled at final positions with a fade. |
| **Graph re-warm (significant mutation, D-03)** | Live render during re-warm: 10–30 ticks over ~150–500ms visible motion, `alpha(0.3)` injected | User sees existing nodes shift and new nodes slide into place. No fade-in per node — continuous live simulation render during the re-warm window. |
| **Graph re-warm (pin/drag)** | Live render, `alpha(0.3)`, ~10–20 ticks, <300ms | Same as above but triggered by `pinNode`. Stop when `alpha < alphaMin`. |
| **Comet head travel (D-14)** | **400ms**, `ease-out-cubic` (`t' = 1 - (1-t)³`) | Head position animates from source node to target node (along edge if one exists, else straight line). |
| **Comet tail fade (D-16)** | 100% opacity for first 2000ms after trail spawn, then linear fade 100% → 0% over remaining 8000ms, total lifespan 10,000ms | Per-segment alpha computed per frame. Evict trails with `age > 10_000ms` or when per-agent cap (10) exceeded (FIFO). |
| **Agent dot pulse (D-17)** | 2s cycle, 2 concentric rings, ring 2 delayed 0.5s, scale 1→~3×, opacity 0.3→0 and 0.2→0 | Matches existing `drawAgentDots` behavior from current `RadarCanvas.tsx:419-468`. Port verbatim. Stops pulsing when agent is idle (no events in last 30s). |
| **Conflict pulse ring (D-22)** | 1.6s cycle, single expanding ring from 1.0× → 2.5× node radius, opacity 1.0 → 0, `cubic-bezier(0, 0, 0.2, 1)`, loops while conflict active | One-color variant using `error` token. Matches existing Phase 5 pattern intent. |
| **Folder hull fade-in** | 300ms opacity fade (0 → 100%) when a hull first appears in render (e.g. after re-warm introduces a new folder cluster) | Prevents hull "pop" on mutation. Hulls that were already rendered in the last frame are not re-faded. |
| **Node hover grow** | 120ms `cubic-bezier(0.2, 0, 0, 1)` radius 5 → 6px and fill surface-container → surface-container-high | Reverse on mouse-leave, same timing. |
| **Node selected outline fade** | 200ms opacity 0 → 80% on white outer stroke | Triggered when `selectedAgentId` changes and the selected agent's current-position file differs from the previous. |
| **Selected-agent ambient glow (inherited)** | 2s pulse: opacity 0.1 → 0.15 via sin, 40px radial blur | Ported verbatim from Phase 4 `drawAgentHighlight`. |
| **Pan/zoom** | Instant (no easing), driven by `useCanvasZoomPan` | Preserved from Phase 4. No inertia in v1. |
| **Drag-to-pin lift** | Node radius grows 5 → 7px over 100ms on `mousedown`, shadow halo appears at 20% agent-palette-color alpha | Communicates "lifted" state during drag. |
| **Drop snap** | Node returns to 5px radius over 150ms, halo fades | On `mouseup`. Position stays at drop point; `node.fx` and `node.fy` set; re-warm with `alpha(0.3)` triggered. |
| **Pinned-node lock badge fade-in** | 150ms opacity 0 → 100% | After drop snap completes. |
| **Manifest collapse (inherited)** | 200ms width 280 → 0px ease-in-out (Motion library) | Preserved from Phase 4. |
| **Mini chat, new request pulse, diff enter, lead-line fade** | Inherited from Phase 4 | Not re-specified. |

### Motion library constraint (DO NOT cross)

**Motion (Framer) is NOT used for Canvas animations.** The comet head, tail fade, pulse, conflict ring, hover, and drag lift are all rendered inside the Canvas `requestAnimationFrame` loop driven by `performance.now()` timestamps — never by a `motion` component. Motion is only used for:

- Manifest panel collapse slide (HTML width animation, existing Phase 4 use)
- Empty-state / banner enter/exit (HTML opacity fade)
- MiniChat expand (HTML, Phase 4)

This is intentional and locked per CONTEXT.md additional-context constraint: "The comet animation runs in the Canvas render loop, not Motion."

## Interaction Contracts

### Pan / zoom

- **Mouse wheel:** zoom in/out, 0.3× – 20× range (slightly wider than Phase 4's 0.5–20× to allow zooming out to see the full graph bounding box). Zoom pivots on cursor position. Preserved hook: `useCanvasZoomPan`.
- **Click + drag on empty canvas:** pan viewport. Cursor: `grab` default, `grabbing` during drag.
- **Double-click on empty canvas:** fit-to-extents — compute `nodes` bounding box, animate viewport over 300ms to frame the whole graph with 10% padding. (New in Phase 7; useful for "how big is this repo?")

### Node hover

- Cursor: `pointer` over a node (quadtree hit-test at `find(world.x, world.y, 8)`).
- Node fill transitions from `surface-container` → `surface-container-high` over 120ms.
- Node radius grows 5 → 6px world-space.
- After 300ms dwell, `AgentTooltip` appears (or a **NodeTooltip** variant if the hovered node has no agent on it). The tooltip reuses the Phase 4 `AgentTooltip` glassmorphism container. Content:
  - If a file with no agent currently on it: file path (Data), `{in_degree}_imports` / `{out_degree}_imported_by` (Data-sm), contention score if > 0 (Data-sm in `error` color).
  - If a file with an agent on it: previous Phase 4 content (agent ID, status, intent) plus the file path.
- Tooltip position: offset +12px right, +12px down from cursor; flips to left/up at viewport edges.

### Node click (file selection)

- Single-click a node with no agent: **no-op** in v1 (no file-detail flyout yet). Reserved for future.
- Single-click a node that has an agent on it: calls `selectAgent(agentId)`. Same behavior as selecting the agent from the manifest — manifest scrolls to and highlights the row; canvas applies the ambient glow.
- Agent's current-position node shows the selected-agent glow + 1px white outer stroke.

### Edge hover

- **Not interactive in v1.** Edges do not intercept pointer events. Reserved tokens exist for future edge hover state but are not wired.

### Drag-to-pin (D-03 re-warm trigger b)

- **Press:** `mousedown` on a node → the node is "picked up."
  - Cursor: `grabbing`.
  - Node radius grows 5 → 7px over 100ms.
  - Agent-palette-colored shadow halo appears at 20% alpha behind the node (indicates lift).
  - Pan is disabled while dragging a node.
- **Drag:** node `x/y` follows cursor in world coordinates. Other nodes do not move during drag.
- **Release:** `mouseup` → node's `fx`, `fy` fixed at drop position. Lock badge appears with 150ms fade-in. Simulation re-warmed with `alpha(0.3)` for up to 300ms (live render). `pinnedNodeIds.add(nodeId)` in store.
- **Unpin:** Shift+click a pinned node → `fx = fy = null`, lock badge fades out over 150ms, simulation re-warmed with `alpha(0.2)`.

### Manifest panel (preserved D-21)

- Click an agent row in the manifest: `selectAgent(agentId)` → canvas ambient glow on that agent's current-position node; viewport animates to center on the node over 300ms.
- Click the selected agent again: deselect; glow fades.
- Manifest collapse toggle: 200ms width slide (Motion). Minimap shifts 12px ↔ 292px right.

### Heat-map toggle (preserved Phase 4)

- Click "HEAT_MAP" button in bottom-left area: toggle `heatMapEnabled`.
- Active state: primary color + primary/10 bg. Node fills switch to `heatColor(score)`. Heat-map tint is OFF by default.

### Minimap

- Shows graph bounding box scaled to 160×120px with 2px padding.
- Nodes shown as 2px dots (heat-tinted if toggle on).
- Viewport rectangle stroked in `primary` (1px) indicating current main-canvas viewport.
- Click inside minimap: main canvas pan animates to the clicked world position over 200ms.
- Drag inside minimap: live pan follows cursor world position (while mouse down).
- Hover minimap: minimap bg opacity rises from 80% → 100% (50ms).

### Conflict pulse (D-22)

- Triggered automatically by `conflictStore` subscription: any file path in `conflictStore.activeConflicts` gets its corresponding graph node pulsing red via the conflict ring animation.
- Conflict badge dot is always rendered on contended nodes regardless of zoom.
- Hover on a conflicted node: tooltip shows `CONFLICT_ACTIVE` label + agent IDs involved (Data-sm, error color).

### Keyboard shortcuts

- Inherited global: `Ctrl+K` command palette.
- Within Radar view (new in Phase 7):
  - `0` or `F`: fit-to-extents (same as double-click empty canvas).
  - `H`: toggle heat-map.
  - `M`: toggle manifest panel.
  - `Escape`: deselect agent and close tooltip.
- No focus-style keyboard navigation between nodes in v1 (accessibility note below).

### Pipeline-driven trail spawn (D-14, D-15, D-18)

- On each `FileEvent` attributed to an agent (from `pipelineStore.events`), check if the agent's previous most-recent file path differs from the new one.
- If yes: push an `ActiveTrail { agentId, fromPath, toPath, startTs: now }` into `radarStore.activeTrails`.
  - Compute trail curve: if an edge exists between `fromPath` and `toPath` node positions, follow the edge. Else, straight line.
  - Enforce per-agent FIFO cap of 10: evict oldest if at cap.
  - Evict all trails with `age > 10_000ms` each frame (`pruneTrails` called from rAF loop).
- Agent's current-position dot snaps to `toPath` after 400ms (coinciding with comet arrival).

All HTML UI chrome (banners, tooltips, minimap) retains the Phase 1 contrast compliance verified by `src/__tests__/theme.test.ts`.

### Keyboard navigation

- Heat-map toggle button: tab-reachable, `aria-label="Toggle heat map overlay"`, `aria-pressed`.
- Manifest collapse toggle: tab-reachable.
- Empty-state CTA (`OPEN_REPOSITORY`): tab-reachable.
- Performance banner dismiss: tab-reachable.
- **Graph nodes: not tab-reachable in v1.** The visual graph is not keyboard-navigable; the manifest panel provides keyboard access to agents (and therefore to their current-position files). This is a known accessibility gap tracked for Phase 8+. Screen reader users receive the manifest list as the primary navigation model.
- **Keyboard hotkeys (new, documented above):** `F` (fit), `H` (heat-map), `M` (manifest), `Escape` (deselect). All must have visible tooltip hint via the command palette.

### Screen reader semantics

- Canvas element: `role="img"` with `aria-label="Codebase dependency graph. {N} files, {E} edges. Use the agent manifest panel for keyboard access to agents and their files."`
- Performance banner: `role="alert"` (overload variant only) / `role="status"` (degraded variant).
- Empty-state overlay: announced as a heading + body text with `aria-live="polite"`.

### Reduced motion (`@media (prefers-reduced-motion: reduce)`)

MUST be honored. Behavior:

- **Comet head travel:** skip animation. New trails snap instantly to target; no tail fade — render a single static line at 40% opacity for 3 seconds, then remove.
- **Agent dot pulse:** render static dot (no pulse rings).
- **Conflict pulse ring:** render static `error`-colored outline at 100% opacity (no expansion).
- **Selected-agent ambient glow:** static 15% opacity (no sin pulse).
- **Folder hull fade-in, node hover grow, drag-lift, drop-snap:** instant state changes, no tweens.
- **Graph settle fade-in, re-warm live render:** still render — these are informational (the user needs to see the graph settle). But cap ticks per frame to avoid over-rendering.
- **Banner transitions:** instant appear/dismiss.

Detection: read `window.matchMedia('(prefers-reduced-motion: reduce)').matches` at `useGraphLayout` init; subscribe to the MediaQueryList `change` event so toggling OS setting hot-updates without reload.

### Contrast-safe per-agent color fallback

Palette index 7 (`#a3e635` lime) and index 0 (`#8eff71` primary) are close in hue. For colorblind users, the manifest row adjacent to each agent dot shows the agent ID in JetBrains Mono Data font — color is never the sole differentiator in v1. Deuteranopia-specific palette substitution deferred to Phase 8+.

### Single-child directory chain collapse (commit `a8fe89b`, RESEARCH Pitfall 7)

Applied to both:
- **Graph node `dirKey` assignment:** a node's `dirKey` is the nearest directory that either has ≥ 2 children OR contains at least one file child. This merges single-child chains like `src/views/Radar/` into one `dirKey`.
- **Folder hull + label:** labels use slash-joined segments, e.g. `VIEWS/RADAR` rather than three nested hulls.

### Repo-relative path convention (commit `a1b15b6`)

- All node `id` values are repo-relative forward-slash paths.
- All edge `from` and `to` values are repo-relative forward-slash paths.
- Backend command `get_dependency_graph` must emit paths via `strip_prefix(repo_root)` + backslash normalization (same helper used by `get_tree_index`, see RESEARCH Pitfall 8).
