**3. Wire contention score updates in `src/views/RadarView.tsx`:**

Add to RadarView:
```typescript
import { useConflictStore } from '../stores/conflictStore';
const alerts = useConflictStore((s) => s.alerts);
const updateContentionScores = useRadarStore((s) => s.updateContentionScores);
const events = usePipelineStore((s) => s.events);
```

Set up a periodic contention score update (every 5 seconds per RESEARCH.md Pitfall 3):
```typescript
useEffect(() => {
  const interval = setInterval(() => {
    // Build agent file events map from pipeline events
    const agentFileEvents = new Map<string, string[]>();
    for (const ev of events) {
      if (ev.attribution.kind === 'pid') {
        const key = `PID-${ev.attribution.value}`;
        const existing = agentFileEvents.get(key) ?? [];
        existing.push(ev.path);
        agentFileEvents.set(key, existing);
      }
    }
    updateContentionScores(alerts, agentFileEvents);
  }, 5000);
  return () => clearInterval(interval);
}, [alerts, events, updateContentionScores]);
```

Also trigger an immediate update when alerts change:
```typescript
useEffect(() => {
  // Immediate update when conflicts change
  const agentFileEvents = new Map<string, string[]>();
  // ... same mapping logic
  updateContentionScores(alerts, agentFileEvents);
}, [alerts]);
```
