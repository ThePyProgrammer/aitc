**1. Extend `src/stores/conflictStore.ts`:**

Add merge resolution state to the existing store interface. Keep ALL existing fields and actions intact.

Add new interface members:
```typescript
// Merge state
activeMerge: {
  conflictId: string;
  filePath: string;
  baseContent: string;
  agentAContent: string;
  agentBContent: string;
  agentAId: string;
  agentBId: string;
  hunks: MergeHunk[];
  resolutions: Map<number, 'a' | 'b' | 'custom'>;
  customEdits: Map<number, string>;
  status: 'loading' | 'diffing' | 'resolving' | 'committing' | 'done' | 'error';
  error: string | null;
} | null;
```

Add new actions:
- `openMerge(conflictId: string)`: Sets status to 'loading'. Calls `invoke<ConflictFileVersions>('read_conflict_files', { conflictId })`. On success, calls `computeMerge(base, agentA, agentB)` to produce hunks. Sets activeMerge with all data, status 'resolving'. On error, sets status 'error' with error message.

- `resolveHunk(hunkIndex: number, choice: 'a' | 'b' | 'custom', customContent?: string)`: Updates `activeMerge.resolutions.set(hunkIndex, choice)`. If choice is 'custom', also `activeMerge.customEdits.set(hunkIndex, customContent!)`. Uses immer-style update (create new Map copies for immutability).

- `applyResolution()`: Sets status 'committing'. Calls `buildMergedContent(hunks, resolutions, customEdits)`. Determines resolutionType: if all resolutions are 'a' -> 'accept_a', all 'b' -> 'accept_b', all 'custom' -> 'manual', mixed -> 'mixed'. Calls `invoke('apply_resolution', { conflictId, mergedContent, hunkResolutions: [...], resolutionType })`. On success, sets status 'done'. Emits "conflict-resolved" event. After 2s delay, sets activeMerge to null.

- `discardAll()`: Sets activeMerge to null.

- `unresolvedCount()`: Returns count of conflict hunks without a resolution entry.

- `resolvedCount()`: Returns count of conflict hunks with a resolution entry.

Import `MergeHunk`, `computeMerge`, `buildMergedContent` from `../lib/merge`.

```typescript
import { useConflictStore } from '../stores/conflictStore';
import { MergeView } from './Conflicts/MergeView';

export function ConflictsView() {
  const activeMerge = useConflictStore((s) => s.activeMerge);
  const alerts = useConflictStore((s) => s.alerts);
  const activeAlerts = alerts.filter((a) => !a.dismissed);
  const openMerge = useConflictStore((s) => s.openMerge);

  // If merge is active, show MergeView
  if (activeMerge) {
    return <MergeView />;
  }

  // If no active conflicts, show empty state (preserve existing empty state)
  if (activeAlerts.length === 0) {
    return (/* existing ZERO_CONFLICTS_DETECTED empty state JSX */);
  }

  // Conflict list: show active alerts as clickable rows
  return (
    <div className="min-h-[calc(100vh-56px)] bg-surface p-6" style={{ animation: 'phosphor-in 150ms ease' }}>
      <h2 className="text-on-surface-variant font-headline text-sm font-bold uppercase tracking-widest mb-4">
        ACTIVE_CONFLICTS
      </h2>
      <div className="flex flex-col gap-2">
        {activeAlerts.map((alert) => (
          <button
            key={alert.id}
            onClick={() => openMerge(alert.id)}
            className="flex items-center gap-4 p-4 bg-surface-container hover:bg-surface-container-high transition-colors duration-150 text-left"
          >
            <span className="text-error font-mono text-xs">{alert.id}</span>
            <span className="text-on-surface font-mono text-sm flex-1 truncate">{alert.filePath}</span>
            <span className="text-on-surface-variant font-mono text-xs">
              {alert.agentAId} vs {alert.agentBId}
            </span>
          </button>
        ))}
      </div>
    </div>
  );
}
```
