|-------|-------|-------|

## Interaction Contracts

### Merge Resolution Flow

1. User clicks a conflict alert (from ConflictsView list or ConflictNavBadge) -- navigates to MergeView
2. MergeView loads file contents from Rust backend (base via git, Agent A version, Agent B version)
3. `node-diff3` computes hunks, displayed as unified diff with syntax highlighting (Shiki)
4. Conflict hunks show Agent A lines (primary/10 background) and Agent B lines (secondary/10 background)
5. Each conflict hunk displays inline resolution controls: `[ACCEPT_A]` ghost button, `[ACCEPT_B]` ghost button, `[EDIT]` ghost button
6. Clicking Accept A/B resolves that hunk immediately -- hunk background changes to resolved green (primary/5), controls disappear, checkmark icon appears
7. Clicking Edit opens an inline textarea within the hunk for manual editing, with a `[SAVE]` primary button
8. Hunk Navigator sidebar updates resolved/unresolved counts per hunk
9. ResolutionToolbar shows progress: "3/5 HUNKS_RESOLVED"
10. When all hunks resolved, "APPLY_RESOLUTION" button becomes enabled (was disabled/dimmed)
11. Clicking APPLY_RESOLUTION writes merged file to disk, creates backup, records resolution in DB
12. Success state: toolbar shows "RESOLUTION_APPLIED" with timestamp, navigates back to conflict list after 2s

### Hunk Navigator Interaction

- Each hunk listed as: `HUNK_01`, `HUNK_02`, etc. in monospace label style
- Unresolved hunks: amber (`#ffd16f`) left border
- Resolved hunks: green (`#8eff71`) left border + checkmark icon
- Click scrolls unified diff to that hunk
- Active hunk (currently visible in viewport): `surface-container-high` background

### History Table Interaction

- Click row to expand inline details (smooth height animation via Motion, 200ms)
- Sessions expanded: shows top 10 most-touched files with write counts
- Conflicts expanded: shows hunk resolution summary, backup file links, agent intent
- Approvals expanded: shows full request details, diff preview, response message
- Sort by clicking column headers (arrow indicator, ascending/descending toggle)
- Filter controls: dropdown for agent, date range picker (ghost button style), status multi-select

### Heat Map Interaction

- Toggle ON: heat map overlay renders over treemap cells with 200ms fade-in (Motion)
- Toggle OFF: overlay fades out 150ms
- Hovering a heated cell shows tooltip with: file path, contention score, conflict count, agent count
- No click interaction on heat map cells (they are informational overlay only)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | APPLY_RESOLUTION |
| Secondary CTA | DISCARD_ALL |
| Empty state heading (Conflicts) | ZERO_CONFLICTS_DETECTED |
| Empty state body (Conflicts) | Airspace clear. File conflicts between agents will surface here. |
| Empty state heading (History) | NO_RECORDS_FOUND |
| Empty state body (History) | No session, conflict, or approval records match your filters. Adjust filters or wait for agent activity. |
| Error state (merge load) | MERGE_LOAD_FAILED -- Unable to read file versions. Verify the file exists and the repository is accessible. |
| Error state (resolution write) | RESOLUTION_WRITE_FAILED -- Could not write merged file to disk. Check file permissions and try again. |
| Error state (file too large) | FILE_EXCEEDS_LIMIT -- File exceeds 1MB inline merge limit. Open in external editor for manual resolution. |
| Destructive: Discard All | DISCARD_ALL: This will discard all hunk resolutions and reset the merge. Unresolved changes will be lost. Confirm? |
| Destructive: Revert Resolution | REVERT_RESOLUTION: This will restore the pre-resolution file state from backup. The current file will be overwritten. Confirm? |
| Intent fallback | No intent available for this agent. |
| Heat map toggle label | HEAT_MAP |
| Hunk resolution labels | ACCEPT_AGENT_A / ACCEPT_AGENT_B / EDIT_MANUAL |
| Resolution progress | {resolved}/{total} HUNKS_RESOLVED |
| Tab labels | SESSIONS / CONFLICTS / APPROVALS |

All copy uses UPPER_SNAKE_CASE for labels and headings per Command Horizon convention. Descriptive body text uses sentence case.

## Accessibility

| Concern | Approach |
|---------|----------|
| Keyboard navigation | Tab through hunk resolution controls; Enter to accept; Escape to cancel edit mode |
| Screen reader | Hunk status announced via aria-live region when resolved; table rows use aria-expanded |
| Color contrast | All text meets WCAG AA (4.5:1) against surface backgrounds; heat map is supplementary (not sole information channel) |
| Reduced motion | Respect `prefers-reduced-motion`: skip phosphor-in, instant heat map toggle, no pulse animations |
| Focus indicators | 2px `primary` outline on focus-visible for all interactive elements |
