**2. Create `src/lib/merge.ts`:**

Export the `MergeHunk` interface:
```typescript
export interface MergeHunk {
  index: number;
  type: 'conflict' | 'clean';
  baseLines: string[];
  aLines: string[];
  bLines: string[];
  startLine: number;
  endLine: number;
}
```

Export `computeMerge(base: string, agentA: string, agentB: string): MergeHunk[]`:
- Split all three inputs by `\n`
- Call `diff3Merge(aLines, baseLines, bLines)` from `node-diff3`
- Iterate results: regions with `ok` property are clean hunks (type: 'clean'), regions with `conflict` property are conflict hunks (type: 'conflict') with `conflict.o` (base), `conflict.a` (agent A), `conflict.b` (agent B)
- Track `lineIndex` to set startLine/endLine
- Return array of MergeHunk

Export `buildMergedContent(hunks: MergeHunk[], resolutions: Map<number, 'a' | 'b' | 'custom'>, customEdits: Map<number, string>): string`:
- For each hunk: if clean, use aLines (same as bLines). If conflict, check resolution map: 'a' uses aLines, 'b' uses bLines, 'custom' uses customEdits.get(hunk.index).split('\n'). If no resolution yet for a conflict hunk, use baseLines.
- Join all lines with `\n` and return.

**3. Create `src/lib/contention.ts`:**

Export `computeContentionScore(conflictCount: number, writeAgentCount: number, maxConflicts: number, maxAgents: number): number`:
- `conflictNorm = maxConflicts > 0 ? conflictCount / maxConflicts : 0`
- `writeNorm = maxAgents > 1 ? (writeAgentCount - 1) / (maxAgents - 1) : 0`
- Return `Math.min(1.0, conflictNorm * 0.7 + writeNorm * 0.3)`

Export `contentionToColor(score: number): string`:
- score 0.0-0.3: `rgba(142, 255, 113, ${score / 0.3 * 0.25})` (green, Command Horizon primary)
- score 0.3-0.7: `rgba(255, 209, 111, ${0.15 + ((score - 0.3) / 0.4) * 0.2})` (amber, Command Horizon tertiary)
- score 0.7-1.0: `rgba(255, 115, 81, ${0.2 + ((score - 0.7) / 0.3) * 0.25})` (red, Command Horizon error)
- score 0: return `rgba(0, 0, 0, 0)` (transparent, no overlay)

**1. Create `src/hooks/useSyntaxHighlight.ts`:**

Implement a singleton Shiki highlighter:

```typescript
import { createHighlighterCore } from 'shiki/core';
import { createJavaScriptRegExpEngine } from '@shikijs/engine-javascript';
import { useState, useEffect } from 'react';

let highlighterPromise: Promise<any> | null = null;

function getHighlighter() {
  if (!highlighterPromise) {
    highlighterPromise = createHighlighterCore({
      engine: createJavaScriptRegExpEngine(),
      themes: [import('shiki/themes/github-dark.mjs')],
      langs: [
        import('shiki/langs/typescript.mjs'),
        import('shiki/langs/javascript.mjs'),
        import('shiki/langs/rust.mjs'),
        import('shiki/langs/json.mjs'),
        import('shiki/langs/css.mjs'),
        import('shiki/langs/html.mjs'),
        import('shiki/langs/python.mjs'),
      ],
    });
  }
  return highlighterPromise;
}
```

Export `useSyntaxHighlight()` hook:
- Returns `{ highlighter, isLoading }`.
- Uses `useState` + `useEffect` to initialize the highlighter on first mount.
- `highlighter` is the resolved Shiki instance or null while loading.
- `isLoading` is true until the highlighter is ready.

Export a `highlightLines(highlighter: any, code: string, lang: string): string[]` helper:
- Calls `highlighter.codeToTokens(code, { lang, theme: 'github-dark' })`.
- Maps each token line to an HTML string of `<span style="color: {color}">{text}</span>` segments.
- Returns array of HTML strings (one per line), NOT using `codeToHtml` which wraps in `<pre>` tags.

Language detection: export `detectLanguage(filePath: string): string`:
- Map file extensions to Shiki language IDs: `.ts`/`.tsx` -> `typescript`, `.js`/`.jsx` -> `javascript`, `.rs` -> `rust`, `.json` -> `json`, `.css` -> `css`, `.html` -> `html`, `.py` -> `python`, default -> `text` (no highlighting).
