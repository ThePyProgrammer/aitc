<interfaces>
JSON-RPC surface (shape; Plan 03 implements):

`initialize`:
- Request: `{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"claude","version":"2.x"}}}`
- Response body: `{"jsonrpc":"2.0","id":1,"result":{"protocolVersion":"2025-03-26","capabilities":{"tools":{}},"serverInfo":{"name":"aitc-chat","version":"0.1"}}}`
- Response header: `Mcp-Session-Id: <uuid-server-generated>`

`tools/list`:
- Request: `{"jsonrpc":"2.0","id":2,"method":"tools/list"}` with `Mcp-Session-Id: <...>`
- Response: `{"jsonrpc":"2.0","id":2,"result":{"tools":[{name,description,inputSchema}, ...]}}`

`tools/call`:
- Request: `{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"get_pending_user_messages","arguments":{}}}`
- Response: `{"jsonrpc":"2.0","id":3,"result":{"content":[{"type":"text","text":"..."}],"isError":false}}`

`notifications/initialized`:
- Request (no id): `{"jsonrpc":"2.0","method":"notifications/initialized"}`
- Response: HTTP 202 Accepted with empty body.

Tool schema for `get_pending_user_messages`:
```json
{"type":"object","properties":{},"required":[]}
```
(No args; server knows the session and binds it to an agent_id via the X-AITC-Session header Claude forwards from write_session_mcp_config.)

Tool schema for `request_user_input`:
```json
{"type":"object","properties":{"prompt":{"type":"string"},"default":{"type":"string"}},"required":["prompt"]}
```

MCP config file shape (D-11 — written by `write_session_mcp_config`):
```json
{
  "mcpServers": {
    "aitc-chat": {
      "type": "http",
      "url": "http://127.0.0.1:<AITC_PORT>/mcp",
      "headers": {
        "X-AITC-Session": "<agent_id>"
      }
    }
  }
}
```

The `X-AITC-Session` request header (set by Claude per the config) is how AITC routes an MCP session to the correct agent_id — this is Plan 03's session-binding vector. On `initialize`, inspect this header; set `McpSession.agent_id = Some(header_value)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `/mcp` HTTP route on 127.0.0.1 | Only loopback-reachable; Claude subprocess is the expected client, but any local process can connect |
| `X-AITC-Session` request header | Untrusted; Claude forwards whatever we put in its config, but a rogue local process could send any value |
| MCP tool inputs (`request_user_input.prompt`) | Untrusted text eventually rendered in OS notifications + transcript |
| Per-session MCP config file | Written to cwd/.claude/ — must not overwrite user files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-14 | Spoofing | /mcp requests from arbitrary local processes | mitigate | Bind to 127.0.0.1 only (inherited from self_register). Rate limit shared with /hook (10 req/sec). Document: for V2, could add a Bearer token in the MCP config that clients must echo; v1 accepts loopback-isolation as the trust boundary. |
| T-10-15 | Tampering | JSON-RPC request body | mitigate | `axum::extract::DefaultBodyLimit::max(256 * 1024)` — 256 KiB body cap (smaller than the 2 MiB /hook cap; MCP bodies should never approach this). Reuse the existing `DefaultBodyLimit` layer on the router — it's already configured for /hook; /mcp inherits. |
| T-10-16 | Session Confusion | stale Mcp-Session-Id after AITC restart | mitigate | HashMap<String, McpSession> is in-memory only. On unknown session id, return 404 per MCP 2025-03-26 spec — Claude will re-initialize. Cold AITC restart = always fresh state. |
| T-10-17 | Injection | X-AITC-Session header | mitigate | Treat as an opaque string; pass to `LiveSessionRegistry` as `&str` (no SQL interpolation, no shell exec). On mismatch (no agent with that id), tool call returns JSON-RPC error -32001 "unknown agent" — never panics. |
| T-10-18 | XSS / prompt-injection | `request_user_input.prompt` text | accept | Text is rendered by React (default-escaped) and by tauri-plugin-notification (sanitized by OS). No HTML construction from this text anywhere. |

| T-10-20 | DoS | unbounded McpState.sessions HashMap | mitigate | Reject initialize when `sessions.len() >= MAX_MCP_SESSIONS (= 64)` with JSON-RPC error -32002 "too many sessions". In practice each AITC user will have 1-5 agents simultaneously — 64 is ample headroom. |

streamable_http:
- `initialize` without Mcp-Session-Id header → generate UUIDv4, insert into McpState.sessions with `agent_id = headers.get("X-AITC-Session")`, respond 200 + Mcp-Session-Id header + protocolVersion 2025-03-26 body.
- `initialize` with EXISTING Mcp-Session-Id header → return the existing session's state (idempotent re-init).
- `tools/list` with valid Mcp-Session-Id → return the two-tool array.
- `tools/call {name:"get_pending_user_messages"}` with valid session bound to an agent_id → call tools::call_get_pending_user_messages; return the drained queue as a text content block.
- `tools/call {name:"request_user_input"}` with valid session + agent_id + prompt → call tools::call_request_user_input; dispatch OS notification; return an ack (`{messages: [], awaitingUser: true}` or similar — planner chooses a schema, documents it in the SUMMARY).
- Any request with an UNKNOWN Mcp-Session-Id header → HTTP 404 + empty body (MCP spec mandate).
- `notifications/initialized` → HTTP 202.
- Unknown method → JSON-RPC error -32601 "Method not found".
- Malformed JSON body → JSON-RPC error -32700 "Parse error" + HTTP 400.
- When `McpState.sessions.len() >= MAX_MCP_SESSIONS` on initialize → JSON-RPC error -32002 "too many sessions" (T-10-20).
- GET /mcp → for v1, return HTTP 405 Method Not Allowed (SSE upgrade is optional polish; document and defer to later).
- DELETE /mcp with valid Mcp-Session-Id → remove from HashMap; return 204.

tools::call_get_pending_user_messages:
- Looks up the session's bound agent_id.
- Drains any queued `OutboundFrame`s that haven't yet been written to stdin (this is a secondary transport path per D-08 — rarely used, but the tool must work).
- Returns `{messages: [{id: <event_id>, content: <text>, created_at: <timestamp>}, ...]}` as a JSON text block.
- For the initial implementation the "queued messages" come from a new method on LiveSessionRegistry, `drain_pending_for_agent(agent_id) -> Vec<(i64, String)>`. The stdin writer task is the primary path; this is a fallback so Claude can poll if AITC wants it to. Planner decides: either (a) add the drain method + hook it into outbound somehow, or (b) for v1 return an empty messages list and document: "v1 uses stdin-frame primary; MCP fallback TBD Phase 11". Recommend option (b) — simpler and matches the D-08 "fallback / catch-all" spirit.

tools::call_request_user_input:
- Extract `prompt` + optional `default` from arguments.
- Call `crate::chat_runtime::notifications::dispatch_chat_notification(app_handle, agent_id, prompt, Some(agent_id))`.

- Return `{ok: true, prompt: <prompt>}` — v1 does NOT long-hold. Long-hold is a Phase 11+ polish; v1 is fire-and-forget (the user can reply by typing into ChatInput, and that message flows through send_chat_message_to_agent normally).

```rust
use axum::{
    extract::Extension,
    http::{HeaderMap, StatusCode},
    response::{IntoResponse, Response},
    Json,
};
use std::sync::Arc;

use super::types::*;
use super::{McpState, McpSession, MCP_PROTOCOL_VERSION, AITC_SESSION_HEADER, MCP_SESSION_HEADER, MAX_MCP_SESSIONS};
use crate::chat_runtime::session_registry::LiveSessionRegistry;

fn new_session_id() -> String {
    // Simple UUIDv4 generator using rand. If `uuid` crate isn't in Cargo.toml,
    // fall back to a random-hex-hyphenated shape that satisfies is_valid_uuidv4.
    let mut bytes = [0u8; 16];
    getrandom::getrandom(&mut bytes).expect("getrandom");
    // v4 fixup: set bits per RFC 4122.
    bytes[6] = (bytes[6] & 0x0f) | 0x40;
    bytes[8] = (bytes[8] & 0x3f) | 0x80;
    format!(
        "{:02x}{:02x}{:02x}{:02x}-{:02x}{:02x}-{:02x}{:02x}-{:02x}{:02x}-{:02x}{:02x}{:02x}{:02x}{:02x}{:02x}",
        bytes[0], bytes[1], bytes[2], bytes[3], bytes[4], bytes[5], bytes[6], bytes[7],
        bytes[8], bytes[9], bytes[10], bytes[11], bytes[12], bytes[13], bytes[14], bytes[15],
    )
}

fn session_id_from(headers: &HeaderMap) -> Option<String> {
    headers.get(MCP_SESSION_HEADER).and_then(|v| v.to_str().ok()).map(String::from)
}

fn aitc_session_from(headers: &HeaderMap) -> Option<String> {
    headers.get(AITC_SESSION_HEADER).and_then(|v| v.to_str().ok()).map(String::from)
}

pub async fn mcp_post_handler<R: tauri::Runtime>(
    Extension(state): Extension<Arc<McpState>>,
    Extension(sessions): Extension<Arc<LiveSessionRegistry>>,
    Extension(pool): Extension<sqlx::SqlitePool>,
    Extension(app): Extension<tauri::AppHandle<R>>,
    headers: HeaderMap,
    Json(body): Json<JsonRpcRequest>,
) -> Response {
    // Enforce session presence on all methods EXCEPT initialize.
    let req_session = session_id_from(&headers);
    let id = body.id.clone().unwrap_or(serde_json::Value::Null);

    match body.method.as_str() {
        "initialize" => {
            {
                let g = state.sessions.lock().await;
                if g.len() >= MAX_MCP_SESSIONS {
                    return Json(JsonRpcResponse::error(
                        id, JSONRPC_TOO_MANY_SESSIONS, "too many sessions".into(),
                    )).into_response();
                }
            }
            let sid = req_session.clone().unwrap_or_else(new_session_id);
            let agent_id = aitc_session_from(&headers);
            {
                let mut g = state.sessions.lock().await;
                g.insert(sid.clone(), McpSession {
                    agent_id, initialized: true,
                    created_at_ms: std::time::SystemTime::now()
                        .duration_since(std::time::UNIX_EPOCH).unwrap_or_default().as_millis() as u64,
                });
            }
            let result = serde_json::json!({
                "protocolVersion": MCP_PROTOCOL_VERSION,
                "capabilities": {"tools": {}},
                "serverInfo": {"name": "aitc-chat", "version": "0.1"},
            });
            let mut resp = Json(JsonRpcResponse::success(id, result)).into_response();
            if let Ok(h) = axum::http::HeaderValue::from_str(&sid) {
                resp.headers_mut().insert(MCP_SESSION_HEADER, h);
            }
            resp
        }
        "notifications/initialized" => {
            (StatusCode::ACCEPTED, "").into_response()
        }
        _ => {
            let sid = match req_session {
                Some(s) => s,
                None => return (StatusCode::NOT_FOUND, "").into_response(),
            };
            // Session lookup.
            let agent_id = {
                let g = state.sessions.lock().await;
                match g.get(&sid) {
                    Some(s) => s.agent_id.clone(),
                    None => return (StatusCode::NOT_FOUND, "").into_response(),
                }
            };

            match body.method.as_str() {
                "tools/list" => {
                    let result = serde_json::json!({"tools": super::tools::tool_list_v1()});
                    Json(JsonRpcResponse::success(id, result)).into_response()
                }
                "tools/call" => {
                    let params = &body.params;
                    let name = params.get("name").and_then(|v| v.as_str()).unwrap_or("");
                    let args = params.get("arguments").cloned().unwrap_or(serde_json::json!({}));
                    let agent_id = match agent_id {
                        Some(a) => a,
                        None => return Json(JsonRpcResponse::error(
                            id, JSONRPC_UNKNOWN_AGENT, "no agent bound to this MCP session".into(),
                        )).into_response(),
                    };
                    match name {
                        "get_pending_user_messages" => {
                            match super::tools::call_get_pending_user_messages(&agent_id, &sessions, &pool).await {
                                Ok(result) => Json(JsonRpcResponse::success(id, result)).into_response(),
                                Err(e) => Json(JsonRpcResponse::error(id, JSONRPC_INTERNAL_ERROR, e)).into_response(),
                            }
                        }
                        "request_user_input" => {
                            match super::tools::call_request_user_input(&agent_id, args, &app, &pool).await {
                                Ok(result) => Json(JsonRpcResponse::success(id, result)).into_response(),
                                Err(e) => Json(JsonRpcResponse::error(id, JSONRPC_INTERNAL_ERROR, e)).into_response(),
                            }
                        }
                        _ => Json(JsonRpcResponse::error(id, JSONRPC_METHOD_NOT_FOUND, format!("unknown tool: {name}"))).into_response(),
                    }
                }
                _ => Json(JsonRpcResponse::error(id, JSONRPC_METHOD_NOT_FOUND, format!("unknown method: {}", body.method))).into_response(),
            }
        }
    }
}

pub async fn mcp_get_handler<R: tauri::Runtime>(
    Extension(_state): Extension<Arc<McpState>>,
    Extension(_app): Extension<tauri::AppHandle<R>>,
) -> Response {
    // v1: SSE upgrade not implemented. Return 405. Claude falls back to POST-polling.
    (StatusCode::METHOD_NOT_ALLOWED, "SSE upgrade not supported in aitc-chat v1").into_response()
}

pub async fn mcp_delete_handler<R: tauri::Runtime>(
    Extension(state): Extension<Arc<McpState>>,
    Extension(_app): Extension<tauri::AppHandle<R>>,
    headers: HeaderMap,
) -> Response {
    let sid = match session_id_from(&headers) {
        Some(s) => s,
        None => return (StatusCode::NOT_FOUND, "").into_response(),
    };
    let mut g = state.sessions.lock().await;
    if g.remove(&sid).is_some() {
        (StatusCode::NO_CONTENT, "").into_response()
    } else {
        (StatusCode::NOT_FOUND, "").into_response()
    }
}
```

```rust
pub async fn call_request_user_input<R: tauri::Runtime>(
    agent_id: &str,
    args: serde_json::Value,
    app: &tauri::AppHandle<R>,
    pool: &sqlx::SqlitePool,
) -> Result<serde_json::Value, String> {
    let prompt = args.get("prompt").and_then(|v| v.as_str())
        .ok_or_else(|| "missing prompt argument".to_string())?;
    let default = args.get("default").and_then(|v| v.as_str()).unwrap_or("");

    // Insert a visible system_note marker into the transcript.
    let payload = serde_json::json!({
        "text": format!("request_user_input: {prompt}"),
        "kind": "awaiting_user",
        "prompt": prompt,
        "default": default,
    });
    let _ = crate::db::events::insert_agent_event(
        pool, agent_id, None, "system_note", &payload, None, None, None
    ).await;

    // Fire OS notification via Phase 4 plumbing (Plan 04 wires dispatch_chat_notification).
    crate::chat_runtime::notifications::dispatch_chat_notification(
        app, agent_id,
        &format!("AWAITING_USER — {}", truncate(prompt, 80)),
        Some(agent_id),
    );

    Ok(serde_json::json!({
        "content": [{"type":"text","text": format!("Notification sent. User will respond via the AITC chat UI.")}],
        "isError": false,
    }))
}

fn truncate(s: &str, max: usize) -> String {
    if s.chars().count() <= max { s.to_string() }
    else { format!("{}…", s.chars().take(max.saturating_sub(1)).collect::<String>()) }
}
```

```rust
use std::path::{Path, PathBuf};
use std::io::Write;

const AGENT_ID_ALLOWED: &str = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789_-";

fn agent_id_valid(id: &str) -> bool {
    !id.is_empty()
        && id.len() <= 128
        && id.chars().all(|c| AGENT_ID_ALLOWED.contains(c))
}

pub fn write_session_mcp_config(
    cwd: &Path,
    agent_id: &str,
    aitc_port: u16,
) -> Result<PathBuf, String> {
    if !agent_id_valid(agent_id) {
        return Err(format!("invalid agent_id shape: {agent_id:?}"));
    }
    let dot_claude = cwd.join(".claude");
    std::fs::create_dir_all(&dot_claude).map_err(|e| format!("mkdir .claude: {e}"))?;

    let target = dot_claude.join(format!("aitc-mcp-{agent_id}.json"));
    let body = serde_json::json!({
        "mcpServers": {
            "aitc-chat": {
                "type": "http",
                "url": format!("http://127.0.0.1:{aitc_port}/mcp"),
                "headers": { "X-AITC-Session": agent_id },
            }
        }
    });
    let rendered = serde_json::to_string_pretty(&body)
        .map_err(|e| format!("serialize mcp config: {e}"))?;

    let mut tmp = tempfile::NamedTempFile::new_in(&dot_claude)
        .map_err(|e| format!("tempfile: {e}"))?;
    tmp.write_all(rendered.as_bytes()).map_err(|e| format!("tmp write_all: {e}"))?;
    tmp.flush().map_err(|e| format!("tmp flush: {e}"))?;
    tmp.persist(&target).map_err(|e| format!("persist: {}", e.error))?;

    Ok(target)
}

pub fn delete_session_mcp_config(cwd: &Path, agent_id: &str) -> Result<(), String> {
    if !agent_id_valid(agent_id) { return Ok(()); }  // best-effort
    let target = cwd.join(".claude").join(format!("aitc-mcp-{agent_id}.json"));
    match std::fs::remove_file(&target) {
        Ok(()) => Ok(()),
        Err(e) if e.kind() == std::io::ErrorKind::NotFound => Ok(()),
        Err(e) => Err(format!("remove_file: {e}")),
    }
}
```
