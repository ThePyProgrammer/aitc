Phase 10 `launch_agent` behavior changes:
- The existing `agents::commands::launch_agent` continues to call `adapter.launch(...)`.
- After `adapter.launch` returns `(pid, child)`, the command inspects `adapter.capabilities().chat_duplex`:
  - If `true`: `chat_runtime::launcher::launch_live_session` was already called INSIDE `ClaudeCodeAdapter::launch` (returns a `LaunchLiveSessionResult`); the command extracts the pipes, spawns parser + outbound + supervisor tasks, and registers a `LiveSession` in `LiveSessionRegistry`.
  - If `false`: launch_detached is used as before, and the command spawns a `spawn_raw_capture_tasks(child, agent_id)` that reads stdout + stderr line-by-line and inserts `raw_stdout` / `raw_stderr` events.

However, per Plan 02's signature, `launch_live_session` returns `LaunchLiveSessionResult { pid, child, mcp_config_path }` — different from `launch_detached`'s `(u32, Child)`. **Decision point**: either keep `adapter.launch` returning `(u32, Child)` and have the command layer do the routing (simpler, less trait churn), OR refactor the trait to return an enum `LaunchResult { Live(LaunchLiveSessionResult), Detached((u32, Child)) }`. **Recommend** the first: keep the trait unchanged, have `ClaudeCodeAdapter::launch` construct and register the `LiveSession` internally (it needs access to `LiveSessionRegistry` + pool + app — pass these via `LaunchOptions` extension or via accessing tauri State through a thread-local — messy).

**Cleaner approach**: invert the control flow. `ClaudeCodeAdapter::launch` builds argv, calls `launch_live_session` to get the `Child` with piped stdio, but then does NOT take the pipes itself. It returns `(pid, child)`. The `agents::commands::launch_agent` Tauri command — which has all the relevant Tauri State (LiveSessionRegistry, pool, app_handle) — inspects `adapter.capabilities().chat_duplex`, and if true: `let mut child = returned_child; let stdin = child.stdin.take(); let stdout = child.stdout.take(); let stderr = child.stderr.take();` then spawns parser + outbound + supervisor. Register the session.

This means: **the adapter still returns `(pid, Child)`, but for `chat_duplex=true` adapters the Child has piped stdio that the command layer consumes.** The adapter's job is just choosing between `launch_detached` (stdin=null) vs `launch_live_session` (stdin=piped). Document this contract in the trait docstring.

use tauri_plugin_notification::NotificationExt;

pub fn dispatch_chat_notification<R: tauri::Runtime>(
    app_handle: &tauri::AppHandle<R>,
    agent_id: &str,
    body: &str,
    deeplink_agent: Option<&str>,
) {
    let title = format!("AWAITING_USER — {agent_id}");

        Ok(Ok(())) => {}
        Ok(Err(e)) => tracing::warn!(agent_id = %agent_id, err = %e, "chat notification send failed"),
        Err(_) => tracing::debug!(agent_id = %agent_id, "notification plugin unavailable (likely test runtime)"),
    }
}

fn spawn_raw_capture_tasks<R: tauri::Runtime>(
    mut child: tokio::process::Child,
    agent_id: String,
    pool: sqlx::SqlitePool,
    app: tauri::AppHandle<R>,
) {
    use tokio::io::AsyncBufReadExt;
    if let Some(stdout) = child.stdout.take() {
        let pool = pool.clone();
        let app = app.clone();
        let agent_id = agent_id.clone();
        tokio::spawn(async move {
            let reader = tokio::io::BufReader::new(stdout);
            let mut lines = reader.lines();
            while let Ok(Some(line)) = lines.next_line().await {
                let payload = serde_json::json!({ "line": line });
                match crate::db::events::insert_agent_event(
                    &pool, &agent_id, None, "raw_stdout", &payload, None, None, None
                ).await {
                    Ok(ev) => { let _ = app.emit("agent-event-appended", &ev); }
                    Err(e) => tracing::warn!(agent_id = %agent_id, err = %e, "raw_stdout insert"),
                }
            }
        });
    }
    if let Some(stderr) = child.stderr.take() {
        let pool = pool.clone();
        let app = app.clone();
        let agent_id = agent_id.clone();
        tokio::spawn(async move {
            let reader = tokio::io::BufReader::new(stderr);
            let mut lines = reader.lines();
            while let Ok(Some(line)) = lines.next_line().await {
                let payload = serde_json::json!({ "line": line });
                match crate::db::events::insert_agent_event(
                    &pool, &agent_id, None, "raw_stderr", &payload, None, None, None
                ).await {
                    Ok(ev) => { let _ = app.emit("agent-event-appended", &ev); }
                    Err(e) => tracing::warn!(agent_id = %agent_id, err = %e, "raw_stderr insert"),
                }
            }
        });
    }
    // Supervisor-ish: wait for exit, emit session_boundary if this is the last subprocess.
    // Low priority — can be added later, for read-only adapters the transcript just stops.
    tokio::spawn(async move {
        let _ = child.wait().await;
    });
}
