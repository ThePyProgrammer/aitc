# Phase 10 — UI Design Contract (CHAT tab inside Communications Hub)

> Visual and interaction contract for the new first-class chat surface that replaces the half-working `ChatThread` / `ChatInput` / `MiniChatCard` buried in Phase 4's `RequestDetail`. Extends the Command Horizon design system established in `src/styles/theme.css` and inherits the Phase 4 Comms Hub UI contract — this is a sibling CHAT tab inside `CommsView`, not a new route.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Command Horizon system, Tailwind v4 `@theme` tokens in `src/styles/theme.css`) |
| Preset | not applicable |
| Component library | none — bespoke components under `src/components/ui/`, `src/components/layout/`, and per-view `src/views/*/` folders |
| Icon library | Lucide React, `strokeWidth={1.5}` (global rule), `size={16}` for inline, `size={20}` for nav/tab chrome |
| Font | Space Grotesk (headlines, labels, tab chrome) + JetBrains Mono (message body, agent IDs, paths, code) — both self-hosted via `src/styles/fonts.css` |

Global rules already in force (inherited from Phase 1 — do not redeclare):
- Zero-radius corners everywhere (`* { border-radius: 0 !important }` in `theme.css`).
- No-Line Rule: borders transparent by default; opt-in via `border-outline-variant/{opacity}` or `border-primary/{opacity}`.
- Body font is JetBrains Mono 14px via `body` selector.
- Default Lucide stroke width is `1.5`.

shadcn gate: skipped. `CLAUDE.md` explicitly rejects shadcn/ui for this project ("Command Horizon is too custom; would gut and restyle every component"). No `components.json` exists and none will be created.

## Typography

Four sizes (14 / 12 / 11 / 10), two weights (400 / 700). Inherited directly from the Phase 4 Comms Hub UI-SPEC and the Phase 9 Arsenal contract — same sizes, same roles, no new scale entries.

| Role | Family | Size | Weight | Line Height | Tracking | Usage in Phase 10 |
|------|--------|------|--------|-------------|----------|-------------------|
| Heading | Space Grotesk | 14px (`text-sm`) | 700 (`font-bold`) | 1.4 | `tracking-widest` (0.1em) uppercase | `COMMUNICATIONS_HUB` view title (inherited), `AGENT_CHANNELS` master-list header |
| Tab / Nav label | Space Grotesk | 11px (`text-[11px]`) | 700 | 1.2 | `tracking-widest` uppercase | `REQUESTS [N]` / `CHAT [N]` tab labels, adapter chip labels (`CLAUDE_CODE` / `CODEX` / `OPENCODE` / `GENERIC`) |
| Label | Space Grotesk | 10px (`text-[10px]`) | 400 | 1.4 | `tracking-widest` uppercase | Session boundary labels (`SESSION_{session_id_prefix} — 14:23 UTC`), `READ-ONLY_TRANSCRIPT` badge, `DELIVERED` / `QUEUED` / `CONSUMED` / `UNSUPPORTED` delivery labels, `STREAMING…` cursor label |
| Body | JetBrains Mono | 14px (`text-sm`) | 400 | 1.5 | normal | Assistant text, user message text, inline code chunks in assistant replies |
| Data | JetBrains Mono | 12px (`text-xs`) | 700 (for IDs) / 400 (for body) | 1.4 | `-0.025em` | Agent IDs in master list, file paths inside tool-use cards, collapsed-tool-use one-line cards, last-event preview in master list |
| Data-sm | JetBrains Mono | 10px (`text-[10px]`) | 400 | 1.4 | `-0.025em` | Event timestamps, session boundary right-side ISO timestamp, per-event `sequence_number` on hover |

Weight budget: 400 (regular) + 700 (bold) only. No 500/600.

Rules (inherited + Phase 10 additions):
- Space Grotesk is always UPPERCASE with wide tracking for labels and tab chrome.
- JetBrains Mono is used for all agent IDs, file paths, code, and message body.
- Never pure white (#FFFFFF) for body text; use `on-surface-variant` (#adaaaa) for chat prose and `on-surface` (#ffffff) only for the currently-focused agent name in master list, the active turn's assistant text, and the user's own outbound messages.
- Assistant text retains JetBrains Mono 14px — consistent with the Phase 4 chat treatment, and mono is semantically correct for agent output that often contains code-adjacent content.
- Inline code spans inside assistant text get `bg-surface-container-lowest` fill + `text-primary` (shiki integration deferred to planner, but the visual contract is primary green on deepest black surface).
- `@user` mentions inside assistant text render in `text-secondary` (#00cffc) bold. This is the chat-side accent twin of the approval-row secondary-for-ask-more-info usage.

**Accent reserved for (explicit list — never decorative, never on arbitrary interactive elements):**

1. Active CHAT tab underline: `border-b-2 border-primary text-primary`.
2. Selected master-list row: `border-l-2 border-primary` + `bg-surface-container-highest` + agent ID becomes `text-primary`. Matches Sidebar NavLink active state and Phase 9 ResourceRow.
3. Unread indicator badge (on CHAT tab label, on COMMS sidebar nav, and on each master-list row): `bg-primary text-on-primary` for 1–99, `bg-primary text-on-primary` with `99+` copy for 100+. Reuses `PendingCountBadge`.
4. Streaming cursor blink: 2px-wide `bg-primary` block, 1s `step-end` blink (reuses `blink-cursor` animation from `src/styles/animations.css`), rendered as the last character of an in-progress `assistant_text` event.
5. Outbound delivery status indicators on user messages:
   - `DELIVERED` → Lucide `Check` in `primary`.
   - `CONSUMED` → Lucide `CheckCheck` (double-tick, mirrors iMessage "read") in `primary`.
   - `QUEUED` → Lucide `Clock` in `tertiary`.
   - `UNSUPPORTED` → Lucide `X` in `error`.
6. Input caret: `caretColor: '#00cffc'` inline style + blink animation (inherited from Phase 4 `ChatInput`).
7. `@user` mentions inside assistant text: `text-secondary font-bold` (unlike approval buttons which use secondary only for ASK_FOR_MORE_INFO, chat uses secondary for the "I need the human" signal).
8. Adapter chip in master list — fixed, semantic color mapping:
   - `CLAUDE_CODE` → `text-primary bg-primary/10 border border-primary/20` (full-duplex tier).
   - `CODEX` / `OPENCODE` / `GENERIC` → `text-tertiary bg-tertiary/10 border border-tertiary/20` (read-only tier).
9. `READ-ONLY_TRANSCRIPT` badge (rendered on both the master-list row and the top of the detail pane for uncapable adapters): tertiary colorway as above.
10. Archived (terminated) agent treatment in master list: the entire row is wrapped with `opacity-50` — color semantics preserved but de-emphasised. Never re-tinted with error or accent.

**Forbidden:**
- No accent on master-list hover (use `bg-surface-container-high` instead).
- No accent on input focus (use `border-outline/20` ghost border; the only accent on the input is the caret itself).
- No accent on session boundaries (use `outline-variant/20` hairline + `on-surface-variant` label).
- No gradients anywhere on the CHAT tab.
- No decorative glow except the inherited primary button hover `shadow-[0_0_10px_rgba(142,255,113,0.4)]` from `Button.tsx`.
- No `text-error` for generic error-state text inside assistant turns (errors bubble through the backend as `system_note` events with a `text-on-surface-variant` label; only `UNSUPPORTED` delivery and `error`-session-boundary genuinely justify error red).

### The No-Line Rule (Inherited)

Boundaries via tonal shifts, not borders. CHAT tab specifically:
1. Master list ↔ Detail pane separation = `bg-surface-container-low` (list) against `bg-surface-container-highest` (detail). Two tier gap = readable without a border.
2. Transcript ↔ Sticky input = `bg-surface-container-highest` (transcript) against `bg-surface-container-lowest` (input). Three tier gap.
3. Event cards separate from page by surface tier, not by border.
4. Ghost borders (`outline-variant/20` at 20% opacity) allowed ONLY for: session boundaries, the input textarea outer frame (already in Phase 4 `ChatInput`), and optionally the transcript scroll-region top/bottom edges if the checker flags contrast issues.

### Errors & edge cases

| Element | Copy |
|---------|------|
| Send failure (outbound transport error) | Inline toast at bottom-right: `MESSAGE_FAILED — {agent_id}` heading, body `{error_message_from_rust}`. 8s auto-dismiss. Uses existing toast pattern from Phase 9 `UndoToast`. |

- Agent IDs displayed verbatim (no case transformation).
- Timestamps: relative within 24h, absolute beyond.

## Interaction Contract

### Tab switching (CommsView)

- Click REQUESTS tab: URL becomes `/comms?tab=requests` (or `/comms` when no param). Renders the existing 3-panel approval layout. Mount animation: `phosphor-in 150ms ease` (inherited).
- Click CHAT tab: URL becomes `/comms?tab=chat`. Renders `ChatView`. Same mount animation.
- `?tab=chat&agent={agent_id}` deep-links directly to that agent's transcript. Used by OS notification click (D-23) and `ApprovalLinkCard` → back-link if ever needed.
- `?tab=requests&request={request_id}` deep-links directly to a pending request. Used by `ApprovalLinkCard` click (chat → approvals navigation). Also used by OS notification for approval events (Phase 8 behavior, preserved).
- Tab state syncs on browser back/forward.

### CHAT master list

- Click agent row: select agent, update URL `?tab=chat&agent={agent_id}`, focus transitions to the transcript. Resets that agent's unread count (backend emits `agent-events-marked-read` on selection).
- Arrow Up/Down while master list has focus: move selection within the list; wraps at ends.
- `Enter` on a row: same as click.
- Hover: `bg-surface-container-high`.
- Selected: `bg-surface-container-highest` + `border-l-2 border-primary` + agent ID becomes `text-primary`.
- Archived section header: collapsible. Persisted open/closed state in `chatStore.archivedCollapsed`. Default = collapsed on first visit, then remembers user preference.
- Default sort: most-recent-activity descending (most-recently-updated agent on top). Per D's "Claude's Discretion", this is locked.
- New event on an unselected agent: that row's unread badge increments. Row reorders to top of its section (Active stays separate from Archived).

### CHAT transcript

- Mount: scrolls to bottom (newest event). No animation on initial load.
- New event arrives while user is at bottom: auto-scrolls to keep the newest event visible. Uses Motion's layout animation for the new card (150ms fade + 4px y-translate, matches Phase 9 toast entrance).
- New event arrives while user is scrolled up: **does not auto-scroll**; instead a floating `↓ {n}_NEW_MESSAGES` pill appears bottom-right of the transcript with `bg-primary/10 text-primary border border-primary/20`. Click the pill to jump to bottom and dismiss.
- Scroll to top: dispatches `loadOlder(agent_id)` to fetch the next older page of events from `agent_events` (D-18 upward infinite scroll).
- Streaming `assistant_text`: card updates in-place as each stream-json chunk arrives. Blinking primary cursor at end. On turn completion, cursor + `STREAMING…` label disappear simultaneously.
- Collapsed `tool_use` card click: toggles expanded state. Uses Motion `layout` + `AnimatePresence` for the expand/collapse (150ms ease, matches MiniChatCard's deprecated expand — we reuse the rhythm but in the new component).
- `ApprovalLinkCard` click: navigates to `?tab=requests&request={id}`.
- Tool-use `→ APPROVAL_{id}` pill click: same — navigates to REQUESTS tab with the request pre-selected.
- Right-click on any event: context menu with `Copy message`, `Copy with metadata` (includes `sequence_number` + `session_id`). Phase 10 ships these two actions only; `Reply-to-thread` and similar deferred to v2.

### Sticky input

- Focus: caret blinks in `secondary`. The `⏎ SEND · ⇧⏎ NEWLINE` hint appears right-aligned inside the textarea container, `text-on-surface-variant/60`.
- Enter: calls `chatStore.sendMessage(agentId, content)`. Message appears immediately in transcript with `deliveryStatus='queued'` (optimistic). Input clears + textarea resets to 40px.
- Shift+Enter: inserts newline. Textarea auto-grows to a max of 120px.
- Ctrl/Cmd+Enter: same as Enter (ergonomic alternative).
- Escape: clears the textarea if non-empty; blurs if already empty.
- Disabled state (read-only adapter): textarea `disabled` + `opacity-50` + `cursor-not-allowed`. Placeholder shows the read-only copy. Send icon stays `text-on-surface-variant/40`. Hover surfaces the tooltip.
- Disabled state (archived agent): same disabled treatment, different placeholder + tooltip copy.
- Outbound delivery status progression (D-10): `queued` → `delivered` (frame written to stdin) → `consumed` (Claude's turn begins against message) OR `unsupported` (frame rejected or adapter has no path). Each status change uses a 150ms color cross-fade on the DeliveryStatus component.

### OS notifications (D-23)

- `@user` mention parse: on each `assistant_text` turn completion, backend checks for `@user` literal token (planner to decide exact regex). On match, fires native OS notification via Phase 4's `dispatch_approval_notification` path with copy from the Copywriting table above.
- Hook/MCP `needs_user_input` signal: same fires, with the prompt text as body.
- Click notification: brings AITC window to foreground, navigates to `/comms?tab=chat&agent={agent_id}` (deep-link via Phase 4's tray-click plumbing, extended to carry the tab+agent query params).
- System tray icon: no change. The tray reflects overall system status (green/amber/red per Phase 4 D-16), not chat unread state — chat unread is deliberately confined to in-app badges per D-22.

### Keyboard (CHAT tab)

| Key | Action |
|-----|--------|
| Ctrl/Cmd+K | Command palette (inherited from Phase 1) |
| `[` / `]` (when CommsView mounted, no input focused) | Cycle between REQUESTS and CHAT tabs |
| Arrow Up/Down (when master list has focus) | Move selection |
| `Enter` (master list) | Select row (same as click) |
| `j` / `k` (when transcript has focus, no input active) | Next / previous event card (vim-style — optional polish; planner can cut if scope tight) |
| `Escape` (input focused with text) | Clear text |
| `Escape` (input focused, empty) | Blur input; focus returns to master list |
| `Ctrl/Cmd+Enter` (input focused) | Send message |
| `/` (global, no modal open) | Focus the currently-visible primary input (master-list filter if we ship one — deferred; otherwise chat textarea) |

### Real-time updates (D-24)

- `chatStore` subscribes to Tauri events: `agent-event-appended`, `agent-turn-started`, `agent-turn-complete`, `agent-session-started`, `agent-session-ended`.
- Capture runs backend-side regardless of which view is active. Unread counts accumulate while the user is on RADAR / TOWER / ARSENAL / HISTORY / CONFLICTS / REQUESTS tab.
- When the user navigates to `?tab=chat&agent={agent_id}`, `chatStore.markRead(agentId)` fires, emits a backend `mark-read` command, resets that agent's unread count to 0.
- If the user is already viewing an agent and a new event arrives, the event appends + unread stays at 0 (they're watching — no badge needed). Confirmed live via `chatStore.selectedAgentId === agentId && isWindowFocused`.

## Accessibility Contract

- Comms tab bar uses `role="tablist"` / `role="tab"` / `aria-selected` / `aria-controls`. Tab switching via Arrow Left / Right when tab bar has focus (standard ARIA pattern). Matches Phase 9 Arsenal contract.
- Master list uses `role="listbox"` + `aria-activedescendant` for keyboard navigation without losing focus.
- Each event card has a sensible heading hierarchy — h3 for per-event author (`"You", "{agent_id}", "System"`) visible only to screen readers (`sr-only`), h4 for event type where applicable (`tool_use` → `Tool use: {tool_name}`).
- Streaming assistant card includes `aria-live="polite"` so screen readers announce incremental chunks without interrupting the user. Completed turns remove `aria-live` to avoid re-announcing.
- Disabled input: `aria-disabled="true"` + the tooltip surface uses `role="tooltip"` with `aria-describedby` binding to the textarea.
- Delivery status icons are decorative (`aria-hidden="true"`); the text label (`DELIVERED` / `QUEUED` / etc.) carries the accessible meaning.
- Color contrast:
  - `on-surface` (#ffffff) on `surface-container` (#1a1919) = ~13:1 (user bubble) — AA.
  - `on-surface-variant` (#adaaaa) on `surface-container-high` (#201f1f) = ~7.4:1 (assistant bubble) — AA.
  - `on-surface-variant` (#adaaaa) on `surface-container-highest` (#262626) = ~6.8:1 (detail pane) — AA.
  - `primary` (#8eff71) on `surface` (#0e0e0e) = ~13:1 (tab underline, cursor) — AAA.
  - `tertiary` (#ffd16f) on `surface-container-high` (#201f1f) = ~8.5:1 (READ-ONLY badge, QUEUED) — AAA.
  - `error` (#ff7351) on `surface-container-high` (#201f1f) = ~5.8:1 (UNSUPPORTED, CONFIRM_CLEAR) — AA.
- Focus-visible: keyboard focus draws a 1px `outline outline-primary` on all interactive elements (inherited Tailwind focus-visible rule with token override in root style). Never relies on background change alone.
- Reduced motion: streaming cursor blink and new-messages-pill pulse both respect `@media (prefers-reduced-motion: reduce)` — cursor becomes a static primary bar, pulse becomes an instantaneous appearance.
