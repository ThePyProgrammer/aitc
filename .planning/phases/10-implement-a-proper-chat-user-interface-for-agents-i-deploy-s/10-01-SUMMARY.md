## Decisions Made

See `key-decisions` in the frontmatter. The core decisions:

1. **Enable `specta`'s `serde_json` cargo feature** so `AgentEvent.payload_json` can stay `serde_json::Value` (matches the plan's `<interfaces>` block verbatim) rather than be downgraded to `String`.
2. **Add minimal `chat_runtime/{mod,types}.rs` in Task 1** — `db/events.rs` imports `crate::chat_runtime::types::AgentEvent`; without a chat_runtime module, `cargo build` at the Task 1 boundary would fail. Task 2 then expands chat_runtime with the full submodule tree. This cross-task coupling is a Rule 3 blocker auto-fix.
3. **`LiveSessionRegistry::is_archived` returns `true` by default** (conservative) so Plan 02 tests are forced to exercise the archived path through `mark_archived` / `bind_session_id` rather than relying on the stub default.
4. **Stable `EMPTY_EVENTS` module-level const in `ChatTranscript`** — vitest caught an infinite-loop when a Zustand selector returned `?? []` (fresh array every render triggers React 19's useSyncExternalStore guard). Documented as a pattern-establishing fix.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocker] Forward-declare `chat_runtime::types::AgentEvent` in Task 1**

- **Found during:** Task 1 (DB migration + events.rs)
- **Issue:** The plan's Task 1 `<action>` block specifies `src-tauri/src/db/events.rs` with `use crate::chat_runtime::types::AgentEvent;`. Task 2 is the one tasked with creating `chat_runtime/`. At the Task 1 commit boundary `cargo build` would fail because the import can't resolve.
- **Fix:** Created a minimal `src-tauri/src/chat_runtime/{mod,types}.rs` containing only the types `db/events.rs` references (AgentEvent, ChatChannel, OutboundFrame, DeliveryUpdate, SessionStartedPayload, SessionEndedPayload, StreamEvent) + `pub mod chat_runtime;` in lib.rs. Task 2 expands `chat_runtime/mod.rs` with the full submodule tree and adds the nine submodule files.
- **Files modified:** `src-tauri/src/chat_runtime/mod.rs`, `src-tauri/src/chat_runtime/types.rs`, `src-tauri/src/lib.rs` (added `pub mod chat_runtime;`).
- **Verification:** `cargo build` passes at Task 1 commit; `cargo test --lib db::events` runs (empty test mod).
- **Committed in:** 66e1c84 (Task 1 commit).

**2. [Rule 3 - Blocker] Enable `specta`'s `serde_json` cargo feature**

- **Found during:** Task 1, first `cargo build` after writing `types.rs`.
- **Issue:** `pub payload_json: serde_json::Value` on `AgentEvent` fails to compile because `specta::Type` isn't implemented for `serde_json::Value` with the default feature set.
- **Fix:** Added `"serde_json"` to the `specta` feature list in `src-tauri/Cargo.toml`. specta 2.0.0-rc.22 exposes this feature; confirmed via `cargo info specta@2.0.0-rc.22`.
- **Files modified:** `src-tauri/Cargo.toml`, `src-tauri/Cargo.lock`.
- **Verification:** `cargo build` passes; `AgentEvent` serialises through tauri-specta bindings (`src/bindings.ts` exports it as `{ payloadJson: JsonValue }`).
- **Committed in:** 66e1c84 (Task 1 commit).

**3. [Rule 1 - Bug] Stable EMPTY_EVENTS sentinel in ChatTranscript selector**

- **Found during:** Task 3, first vitest run on `ChatTranscript.test.tsx`.
- **Issue:** `useChatStore((s) => agentId ? s.eventsByAgent[agentId] ?? [] : [])` returns a *fresh* empty array each call. React 19's useSyncExternalStore `getSnapshot` stability guard detected this and triggered `Maximum update depth exceeded` in 2 of 3 tests (and 2/2 ChatView tests).
- **Fix:** Declare a module-level `const EMPTY_EVENTS: never[] = []` sentinel and return that instead of `[]`. Same reference every render => no phantom diff.
- **Files modified:** `src/components/chat/ChatTranscript.tsx`.
- **Verification:** All 5 ChatTranscript + ChatView tests pass; Arsenal test suite regressions check also clean.
- **Committed in:** 17501eb (Task 3 commit).
