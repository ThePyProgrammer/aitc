<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> Tauri command | Frontend requests source signature metadata from backend. |
| Repo filesystem -> parser | Source files are read and parsed by tree-sitter. |
| Backend DTO -> frontend store | Signature strings cross IPC and later render in DOM. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering / Information Disclosure | get_source_signatures / get_source_snippet | mitigate | Prefer no path argument for signatures; for snippet paths, accept repo-relative paths only, canonicalize under active watched repo root, reject arbitrary absolute paths/`..`, and cap output at 12 lines. |
| T-13-02 | Tampering | SourceSignatureDto strings | transfer | Backend returns strings only; Plan 13-05 must render them as JSX text, not unsanitized HTML. |
| T-13-03 | Denial of Service | extract_source_signatures | mitigate | Reuse `MAX_FILE_SIZE_BYTES = 1_048_576`, `MAX_PARSE_DURATION = 500ms`, parser/query cache, and empty-Vec-on-failure behavior. |
| T-13-04 | Denial of Service | radarStore signature merge | mitigate | Merge once during `fetchGraph`; do not parse or derive signatures in rAF. |
</threat_model>
