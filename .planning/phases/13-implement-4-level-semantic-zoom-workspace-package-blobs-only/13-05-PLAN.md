<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GraphNode signature/source metadata -> React DOM | Source-derived strings render in code preview cards. |
| User click -> expand snippet state | User can reveal read-only snippet content locally. |
| Keyboard input -> overlay/bridge state | Escape changes local overlay state and bridge selection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering / Information Disclosure | CodePreviewOverlay snippets | mitigate | Invoke only the Plan 13-04 repo-scoped snippet command with repo-relative node ids and render backend-capped 12-line responses; never accept arbitrary absolute paths in the UI. |
| T-13-02 | Tampering | CodePreviewOverlay.tsx | mitigate | Render signatures/source as JSX text; if highlighter output is used, only use existing escaped `highlightLines` output and test it. |
| T-13-03 | Denial of Service | CodePreviewOverlay signature list | mitigate | Render at most `6` cards and do not parse source in the browser. |
| T-13-04 | Denial of Service | CodePreviewOverlay / RadarCanvas | mitigate | Cap cards at `6`, select focused subset only, clamp bounds, and keep overlay DOM bounded. |
</threat_model>
