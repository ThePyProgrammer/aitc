## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon/text gaps, conflict badge offsets, card internal micro-gaps |
| sm | 8px | Compact canvas HUD padding, tooltip/card row gaps |
| md | 16px | Default overlay spacing, code-card padding, control separation |
| lg | 24px | Panel-to-canvas spacing and focused card clusters |
| xl | 32px | Large overlay gutters and inspector separation |
| 2xl | 48px | Major radar region breathing room |
| 3xl | 64px | Page-level empty-state spacing |

Exceptions: Canvas world-space primitives may divide by zoom for visual constancy, but their base values must remain multiples of 4 where practical. Existing locked exceptions remain valid: file node radius 5px, hover/selected radius 6px, arrow length 5px, bridge half-diagonal 8px, bridge hit radius 10px, conflict pulse inner 6px/outer 15px. Phase 13 additions use these exact exceptions:

| Exception | Value | Usage |
|-----------|-------|-------|
| Package blob minimum visual diameter | 24px screen-space | Small packages remain selectable at workspace/package levels |
| Package blob maximum visual diameter | 96px screen-space | Large packages do not swallow the radar overview |
| Package blob hit target | max(blob radius, 22px) | 44px minimum mouse target diameter for centroid-attached agents |
| Signature card width | 320px | Focused code-preview card, not a built-in editor |
| Signature card max height | 240px | Prevents code zoom from covering the radar |

## Semantic Zoom Contract

### Levels, bands, and labels

Use the existing zoom anchors from Phase 13 context: `0.6`, `2`, and `4`. Do not change wheel zoom, pan, minimap behavior, viewport clamp, or fit-to-graph behavior.

| Level | Zoom range where dominant | Representation | HUD label |
|-------|---------------------------|----------------|-----------|
| Workspace | `< 0.6` | Top-level package blobs only, bridge spine, aggregated agents/conflicts/heat | WORKSPACE |
| Package | `>= 0.6` and `< 2` | Sub-package blobs plus unlabeled file dots, bridge spine, aggregated agents/conflicts/heat | PACKAGE |
| File | `>= 2` and `< 4` | File dots, file names, edges, arrows, agent indicators, bridge spine | FILE |
| Code | `>= 4` | File representation plus focused signature/code-preview cards | CODE |

Crossfade contract:

- Crossfade around each anchor over a `0.20` zoom-unit band: `[anchor - 0.10, anchor + 0.10]`.
- Opacity interpolation is linear in zoom and clamped to `[0, 1]`.
- Outside the band, the dominant level opacity is exactly `1` and the adjacent level opacity is exactly `0`.
- During crossfade, only the representation with opacity `>= 0.5` handles hover/click. If both are exactly `0.5`, the higher-detail level wins.
- Hit-testing must follow the visible z-order: bridges first, focused code cards second, agents third, active semantic representation fourth.

### Workspace level

- Render top-level package blobs only. Hide file dots, file labels, non-IPC file edges, arrowheads, and raw folder hull labels.
- Package blob size encodes package file count using square-root scaling: `diameter = clamp(24, sqrt(fileCount) * 8, 96)` screen-space pixels.
- Blob fill uses `theme.hullFill`; stroke uses `theme.hullStroke`; selected/hovered blob uses `primary` glow at 10% opacity with a 40px blur.
- Heat/activity aggregates upward: package heat is the maximum child contention score, with recent agent activity adding a warning glow but not overriding conflict red.
- Top-level package labels are always eligible, but only render if projected label width fits inside `blob diameter + 32px` or the package is in the top 8 by file count/activity/conflict score.
- Agent dots attach to the package centroid of their current file. If multiple agents attach to one blob, fan them around the centroid on an 8px screen-space ring.

### Package level

- Render visible sub-package blobs and unlabeled file dots. File labels remain hidden.
- Sub-package blob size uses the same square-root scaling as workspace but clamps to `20px–72px` screen-space.
- Unlabeled file dots use the existing `NODE_RADIUS_DEFAULT` and heat tint behavior.
- Edges remain hidden except IPC bridge edges. This prevents the package level from becoming a hairball.
- Sub-package labels are importance-filtered. Importance score: `fileCount + (conflictCount * 50) + (activeAgentCount * 25) + (contentionScore * 20)`. Render labels for top 12 visible packages and any package with conflictCount > 0.
- Agents crossfade from package centroid positions toward exact file positions across the package→file band around `2`.

### File level

- Render file dots, file labels, edges, arrowheads, selected-node halo, agent dots, trails, conflict pulses, conflict badges, bridge spine, and bridge labels.
- File labels render at zoom `>= 2`, not the previous `>= 4`, because Phase 13 shifts `>= 4` to code inspection. Labels must still be viewport-culled.
- Edge rendering follows existing GraphRenderer tokens. IPC edges preserve their Phase 12 alpha boost and bridge z-order.
- Hover/click target for file nodes uses existing `NODE_HIT_RADIUS / zoom` behavior.
- Agent indicators snap to exact file-node positions when file level opacity is dominant.

### Code level

- Code zoom is an inspection layer, not an editor.
- Render signature cards only for a focused subset: hovered file, selected file, active agent current files in viewport, and files within `160px` screen-space of viewport center. Hard cap: 6 cards visible at once.
- Default card content shows signatures/exported symbols first. Do not show raw source by default.
- Signature data source priority: existing graph/source-scan metadata; then dependency/source scan if already available; then path metadata fallback. Do not introduce a full language indexer for this phase.
- Raw source snippets appear only after `EXPAND_SNIPPET`. Expanded snippets are read-only, capped at 12 lines, and use existing syntax-highlight/code-preview conventions where available.
- Card placement anchors to the file position with a 16px screen-space leader line. Keep the card inside the canvas bounds with 8px inset.
- Card styling: `bg-surface-container-high/90`, `backdrop-filter: blur(20px)`, no rounded corners, ghost border only if needed at `outline-variant/15`, mono text, secondary signal for metadata chips.

## Layering and Interaction Contract

Render order, back to front:

1. Canvas background from active `GraphTheme.canvasBackground`.
2. Workspace/package blob fills and strokes for the active semantic representations.
3. File edges and arrowheads when file/code representation opacity is active.
4. File dots and heat tint.
5. File labels and package labels.
6. Bridge boundary line, bridge diamonds, bridge labels, and boundary anchor labels. Bridges remain visible at every semantic zoom level.
7. Agent trails and agent dots. Agents remain visible at every semantic zoom level.
8. Conflict pulses and conflict badges. Conflicts are always top-priority status signals.
9. Focused code signature cards and leader lines.
10. DOM HUD: zoom value, semantic label, heat-map toggle, performance banners, tooltips.

Interaction rules:

- Wheel zoom and pan behavior are unchanged from Phase 11.1.
- Minimap behavior is unchanged.
- Existing bridge hover/click precedence remains first.
- During crossfade, only the dominant semantic representation handles hover/click.
- Package blob hover shows package name, file count, active agent count, conflict count, and top hot path if present.
- Package blob click focuses/zooms toward package centroid but must not mutate graph layout or pin nodes.
- File click/hover behavior remains compatible with existing selected/hovered node behavior.
- Code card expand/collapse is local UI state and must not create repository edits.
- Escape closes expanded snippets/signature cards first, then preserves existing bridge deselection behavior.

## Performance Contract

- Do not recompute package hierarchy, blob membership, label importance, or aggregate heat per frame. Compute from existing `graphNodes`, `parentChildMap`, `dirsWithOwnFiles`, `contentionScores`, conflict paths, and agent positions with memoized/cache-keyed helpers.
- Package blob geometry may reuse or extend `hullCache`; bridge nodes must stay excluded from package/hull grouping.
- Crossfade opacity calculation is pure arithmetic and safe to run per frame.
- Signature cards are capped at 6 visible cards and must be DOM overlays or cheap Canvas text passes, not per-file render loops over all visible nodes.
- Maintain 10k-file radar expectations: viewport culling stays active; file labels/cards are culled; no full-source preview rendering for every file.
- Phase 14 owns offscreen-canvas architecture. Phase 13 must not rewrite rendering into OffscreenCanvas or WebGL.
