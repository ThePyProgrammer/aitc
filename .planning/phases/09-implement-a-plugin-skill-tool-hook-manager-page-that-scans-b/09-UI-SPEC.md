# Phase 9 — UI Design Contract (ARSENAL)

> Visual and interaction contract for the ARSENAL page — first master/detail layout in the app. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Inherits the Command Horizon design system already established in `src/styles/theme.css`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Command Horizon system, Tailwind v4 `@theme` tokens) |
| Preset | not applicable |
| Component library | none — bespoke components under `src/components/ui/` and per-view `src/views/*/` folders |
| Icon library | Lucide React, `strokeWidth={1.5}`, default `size={20}` for nav, `size={16}` for inline |
| Font | Space Grotesk (headline / nav / labels) + JetBrains Mono (data / paths / IDs / body) — both self-hosted via `src/styles/fonts.css` |

Global rules already in force (do not redeclare in components):
- Zero-radius corners everywhere (`* { border-radius: 0 !important }` in `theme.css`).
- Default borders removed (No-Line Rule); borders are opt-in via `border-outline-variant/{opacity}`.
- Body font is JetBrains Mono at 14px via `body` selector.

## Spacing Scale

Declared values (all multiples of 4, sourced from `@theme --spacing-*` in `theme.css`):

| Token | Value | Usage in ARSENAL |
|-------|-------|------------------|
| xs | 4px | Icon-to-label gaps inside chips and row meta |
| sm | 8px | Row meta separators, tab gaps, toast internal spacing |
| md | 16px | Resource row horizontal padding, detail panel inner gutter |
| lg | 24px | View-level horizontal padding (`px-6`), section headers |
| xl | 32px | Empty-state stack spacing, detail-panel section breaks |
| 2xl | 48px | Reserved — not used on this page |
| 3xl | 64px | Reserved — not used on this page |

Fixed dimensions (master/detail primitive — ARSENAL establishes these for future reuse):

| Region | Width | Rationale |
|--------|-------|-----------|
| Left category rail | 220px fixed (`w-[220px] shrink-0`) | Four static categories with icon + label + count; narrower than Comms rail (280px) because rows are single-line. |
| Center resource list | `flex-1` with `min-w-[420px]` | Must fit resource name + one-line description + GLOBAL/PROJECT chip + truncated path on one row at 1200px viewport. |
| Right detail panel | 520px fixed on ≥1440px viewports; `flex-1` min 420px on smaller (`w-[520px] 2xl:w-[520px] xl:w-[480px] shrink-0`) | Large enough for frontmatter table + CLAUDE.md editor at readable width (~80ch mono). |
| Divider treatment | 1px via `bg-outline-variant/20` on the rail's trailing edge (`border-r` is banned by No-Line Rule; use positioned `div` or `bg` on parent) | Matches existing Comms 3-panel rails which use `bg-surface-container-low` contrast rather than borders. |

Vertical rhythm:
- View root: `h-[calc(100vh-56px)]` (accounts for 56px TopBar) — matches every existing view.
- Scope tab bar: `h-11` (44px) — matches History tab bar.
- Filter input row: `h-12` (48px) — matches History filter bar.
- Resource row: `h-14` (56px) fixed — single-line content, comfortable click target.

Exceptions: none. All values above resolve to multiples of 4.

## Typography

Three sizes, two weights, inherited directly from the Command Horizon system:

| Role | Family | Size | Weight | Line Height | Tracking | Usage in ARSENAL |
|------|--------|------|--------|-------------|----------|------------------|
| Page heading | Space Grotesk | 14px (`text-sm`) | 700 (`font-bold`) | 1.2 | `tracking-widest` (0.1em) uppercase | "ARSENAL" page title + category rail label |

## Color

60/30/10 split mapped to existing `@theme` tokens. Colors below are references — components MUST use the token class (e.g. `bg-surface`), never the hex.

| Role | Token | Hex | Usage in ARSENAL |
|------|-------|-----|------------------|
| Dominant 60% — page background | `surface` | `#0e0e0e` | Main view background, center resource list background |
| Dominant 60% — rail/panel tier | `surface-container-low` | `#131313` | Left category rail + right detail panel surface (one tier up from page, matches Comms RequestQueue) |
| Secondary 30% — row hover / selected | `surface-container` / `surface-container-high` | `#1a1919` / `#201f1f` | Resource row hover = `container`; selected row = `container-high` with left-border accent |
| Secondary 30% — scope chip surface | `surface-container-high` | `#201f1f` | GLOBAL/PROJECT chip background |
| Secondary 30% — text default | `on-surface-variant` | `#adaaaa` | Resource descriptions, paths, frontmatter labels |
| Secondary 30% — text emphasised | `on-surface` | `#ffffff` | Selected resource name, page heading, editor text |
| Secondary 30% — divider | `outline-variant/20` | `#494847` @ 20% | Rail edges, section separators inside detail panel |
| Accent 10% — primary signal | `primary` | `#8eff71` (phosphor green) | Active scope tab underline, selected row left-border, active category icon, Save button, undo-toast accent |
| Accent 10% — info signal | `secondary` | `#00cffc` (cyan) | External-change banner icon + "Reload" button outline |
| Accent 10% — warning signal | `tertiary` | `#ffd16f` (amber) | GLOBAL scope chip text color (PROJECT uses `primary`) — distinguishes global/project at a glance |
| Destructive | `error` | `#ff7351` | Discard edits button, "Keep mine (overwrite)" action text in external-change banner |

Accent reserved for (explicit list — NEVER use primary/secondary/tertiary on arbitrary interactive elements):
1. Active scope tab: `border-b-2 border-primary` + `text-primary` (matches History tab pattern).
2. Selected resource row: `border-l-2 border-primary` + `bg-surface-container-high` + name text `text-primary` (matches Sidebar NavLink active state).
3. Active category in left rail: icon + label in `text-primary`, inactive in `text-on-surface-variant`.
4. Save button on CLAUDE.md editor: `bg-primary text-on-primary` (reuses `<Button variant="primary">`).
5. Undo toast accent stripe: 2px `bg-primary` left edge + countdown ring in `primary/40`.
6. Scope chip: `text-primary` for PROJECT, `text-tertiary` for GLOBAL. Chip background stays `surface-container-high` for both.
7. External-change banner Reload button: `border border-secondary/40 text-secondary`.
8. Discard / "Keep mine" destructive text: `text-error`.

Forbidden:
- No accent on category rail hover states (use `bg-surface-container-high` instead).
- No accent on filter input focus border (use `border-outline/40`).
- No accent on resource row hover (only on selection).
- No gradients. No glows except the existing Button hover `shadow-[0_0_10px_rgba(142,255,113,0.4)]` (already in `Button.tsx`).

### Empty states (per category × scope combination)

When a category has zero resources in the active scope, render the standard Command Horizon empty-state stack: blink-cursor bar + headline + body copy (reuses `CommsView` empty-state visual pattern).

| Context | Headline (Space Grotesk 14px bold uppercase widest) | Body (JetBrains Mono 12px on-surface-variant/60) |
|---------|-----------------------------------------------------|---------------------------------------------------|
| Skills, any scope | `NO_SKILLS_INSTALLED` | `No SKILL.md files found under {scope-path}. Skills will appear here when Claude installs them.` |
| Agents, any scope | `NO_AGENTS_REGISTERED` | `No agent definitions found under {scope-path}/agents/.` |
| Plugins, any scope | `NO_PLUGINS_INSTALLED` | `installed_plugins.json is empty or absent at {scope-path}.` |
| Configuration, any scope | `NO_CONFIGURATION` | `No hooks, commands, settings.json, or MCP servers detected in {scope-path}.` |
| Combined scope, all categories empty | `ARSENAL_EMPTY` | `Claude has no resources registered globally or in this project. Verify ~/.claude/ and <cwd>/.claude/ exist.` |
| No resource selected (detail panel) | `NO_RESOURCE_SELECTED` | `Select a resource from the list to view its definition.` |

Where `{scope-path}` is rendered in JetBrains Mono at `text-primary/60` for emphasis (e.g. `~/.claude/skills/`).

### Action buttons (CLAUDE.md editor)

| Button | Variant | Copy |
|--------|---------|------|
| Save (primary CTA for this phase) | `<Button variant="primary">` | `SAVE` |
| Discard | `<Button variant="ghost">` with `text-error` override | `DISCARD` |
| (when no unsaved changes) Save is disabled | `<Button variant="primary" disabled tooltip="No changes to save">` | `SAVE` |

### Toasts

Undo toast (shown after every successful save):

| Slot | Copy |
|------|------|
| Title | `SAVED — {filename}` (e.g. `SAVED — CLAUDE.md`) |
| Countdown label | `Undo in {n}s` (JetBrains Mono 10px, updates every second) |
| Action | `UNDO` (Space Grotesk 11px bold uppercase widest, `text-primary`) |
| Dismiss | `×` glyph, `text-on-surface-variant` |

Auto-dismiss: 10 seconds. On UNDO, toast is replaced by a second toast: `RESTORED — {filename}` (3s auto-dismiss, no action).

Save error toast: `SAVE_FAILED — {filename}`, body `{error message from Rust}`, 8s auto-dismiss, no action.

### External-change banner (above editor)

Non-blocking, persistent until user acts. Secondary-tier surface (`bg-surface-container-high`) with 2px `border-l border-secondary` accent stripe.

| Slot | Copy |
|------|------|
| Icon | Lucide `AlertCircle` at 16px `text-secondary` |
| Message | `This file changed on disk while you were editing.` |
| Actions (left to right) | `RELOAD` (ghost, `text-secondary`) · `KEEP MINE` (ghost, `text-error`) · `VIEW DIFF` (ghost default) |

### Destructive actions & confirmations

This phase has two user actions that can lose work. Neither uses a modal — instead they use the undo toast pattern (for Save) and the banner pattern (for external change).

| Action | Confirmation approach |
|--------|-----------------------|
| Save CLAUDE.md (writes to disk immediately) | No modal. Commit optimistically, then show undo toast for 10s. Matches D-14. |
| Discard unsaved edits | Inline confirmation: Discard button first click shows its label change to `CONFIRM DISCARD` with `text-error` for 3s; second click within 3s discards. If the 3s window lapses, label reverts. No modal. |
| Keep mine (overwrite external change) | Treated like Discard: two-click confirmation. First click changes label to `CONFIRM OVERWRITE`; second click within 3s proceeds. |
| Reload (discard local edits) | Two-click confirmation if there are unsaved local edits; single-click if the buffer is clean. |

### Read-only indicators

Resources that cannot be edited in v1 (everything except the two whitelisted CLAUDE.md files) must show a lock affordance:

| Element | Treatment |
|---------|-----------|
| Detail panel header for read-only resource | Lucide `Lock` icon 14px `text-on-surface-variant` to the left of the resource name, + tooltip on hover: `Read-only in v1 — {resource-type} editing planned for a future phase.` |
| Read-only CLAUDE.md (global) | Full banner at top of editor region: `READ-ONLY — ~/.claude/CLAUDE.md editing is disabled this phase.` — `bg-surface-container-high`, `text-on-surface-variant`, `Lock` icon. No editor rendered, just pre-formatted viewer using the same JetBrains Mono 14px 1.5 treatment. |

## Interaction Contract (Phase 9-specific — extends the shared design system)

Because ARSENAL is the first master/detail layout, lock these behaviors:

### Keyboard

| Key | Action |
|-----|--------|
| `↑` / `↓` | Move selection within the center resource list (wraps at ends). |
| `Enter` | Focus the detail panel (shift focus out of the list for screen readers). |
| `Esc` (in filter input) | Clear the filter. |
| `Esc` (in editor with unsaved changes) | Trigger Discard confirmation (2-click pattern above). |
| `Esc` (in editor clean) | Blur the editor. |
| `Ctrl/Cmd + S` (in editor) | Save. Fires the same undo toast. |
| `Tab` / `Shift+Tab` | Move between scope tabs when the tab bar has focus. Activates on focus (matches History tab behavior). |
| Global `/` | Focus the per-category filter input (only when ARSENAL is mounted and no modal is open). |

### Mouse

- Single click on a resource row: select + load into detail panel.
- Hover on resource row: `bg-surface-container`.
- Selected row: `bg-surface-container-high` + `border-l-2 border-primary` + name becomes `text-primary`.
- Hover on category rail item: `bg-surface-container-high` on inactive; active item already has `bg-surface-container` + primary text.

### Animations

Reuse existing Command Horizon animations in `src/styles/animations.css`:
- View mount: `animation: 'phosphor-in 150ms ease'` on the view root (matches all existing views).
- Empty-state cursor: `animation: 'blink-cursor 1s step-end infinite'` on a `h-5 w-[2px] bg-secondary` element (matches CommsView empty state).
- Scope tab underline slide: 150ms transition on `border-b`.
- Toast entrance: 150ms opacity + 4px translate-y from below (use Motion's `AnimatePresence`).

No new animation primitives are introduced. No radar pulse on ARSENAL (this is a config page, not a live-monitoring surface).

### Real-time updates (watcher-driven)

When the pipeline emits a resource-event batch that mutates the currently-selected resource:
- If the user has no unsaved edits in the detail panel: silently refresh the detail panel content. No flash.
- If the user HAS unsaved edits and the file on disk changed (see D-15): show the external-change banner above the editor. Do NOT overwrite the buffer.
- When a row is inserted/removed in the list, use `layout` prop on Motion to animate position (reuse pattern from RadarManifest if present; otherwise fall back to CSS transition on `transform`).

## Accessibility Contract

- All icon-only buttons include `aria-label` (matches Sidebar toggle pattern).
- The scope tabs use `role="tablist"` / `role="tab"` / `aria-selected` (History currently does not — ARSENAL ships it correctly and History can retrofit later).
- The resource list uses `role="listbox"` with `aria-activedescendant` for keyboard navigation. Arrow keys move selection without losing focus on the filter input.
- Color contrast: `on-surface` on `surface` is white on `#0e0e0e` (ratio ~17:1). `on-surface-variant` (#adaaaa) on `surface` is ~7.6:1 — passes AA. Scope chip text (`primary`/`tertiary`) on `surface-container-high` both pass AA for 11px bold (~7:1).
- Focus-visible: all interactive elements receive a 1px `outline outline-primary` on keyboard focus (Tailwind default `focus-visible:outline` with token override in a root style). Do not rely on background color change alone.
