<objective>
Build the conflict detection engine that runs in the Rust backend, consuming the attributed file event stream and flagging overlapping writes between different agents within a configurable time window.

Purpose: Per D-10/D-12/CNFL-06, conflict detection must be real-time and backend-powered. The engine uses a per-file sliding window to catch when two different agents write the same file within the conflict window (default 5 seconds). Per D-12, it captures hunk hints (byte ranges/line counts) when available for Phase 5's merge UI.

Output: `conflict` Rust module with engine, types, and Tauri commands.
</objective>

3. Create `src-tauri/src/conflict/engine.rs`:
   - `ConflictEngine` struct:
     - `recent_writes: HashMap<PathBuf, Vec<FileWriteRecord>>` -- per-file sliding window
     - `window: Duration` -- configurable, default 5 seconds
     - `pid_to_agent_id: HashMap<u32, String>` -- cache of PID-to-agent-ID mapping (populated from registry lookups)
   - `ConflictEngine::new(window: Duration) -> Self`
   - `ConflictEngine::set_window(&mut self, window: Duration)` -- update conflict window
   - `ConflictEngine::update_pid_mapping(&mut self, pid: u32, agent_id: String)` -- called when registry learns a PID->agent mapping
   - `ConflictEngine::process_batch(&mut self, batch: &FileEventBatch) -> Vec<ConflictAlert>`:
     - For each event in batch where `event.kind` is `Modify` or `Create` (writes only):
       - Skip if attribution is `Unattributed` or `Ambiguous` (can't identify agent).
       - Extract PID from `Attribution::Pid(pid)`.
       - Look up agent_id from `self.pid_to_agent_id`. If not found, use `format!("PID-{pid}")` as fallback ID.
       - Look up `self.recent_writes.entry(event.path.clone())` and get the file's write records.
       - Evict entries older than `self.window` from the vector (compare `event.timestamp_ms - record.timestamp_ms > window.as_millis()`).
       - Check if any remaining entry has a DIFFERENT agent_id (not just different PID -- per Pitfall 3, same agent different PIDs should also be checked, but different agent_id is the primary check).
       - If conflict found: create ConflictAlert with both agents' data, push to results.
       - Add current write as new FileWriteRecord to the vector.
     - Return collected alerts.
   - `ConflictEngine::sweep_empty_files(&mut self)` -- remove HashMap keys with empty Vec values. Call this every N batches (e.g., every 100) or on a timer to prevent memory growth per Pitfall 6.
   - Add all 10 tests from the behavior list. Use deterministic timestamps (not Utc::now) for reliable testing. Create helper `fn make_batch(events: Vec<(PathBuf, i64, Attribution)>) -> FileEventBatch` for test convenience.
