1. Create `src-tauri/src/agents/launcher.rs`:
   - Constants: `CREATE_NEW_PROCESS_GROUP: u32 = 0x00000200`, `DETACHED_PROCESS: u32 = 0x00000008` (Windows-only, behind `#[cfg(windows)]`).
   - `pub async fn launch_detached(program: &str, args: &[&str], cwd: &Path, env_vars: Option<Vec<(&str, &str)>>) -> Result<(u32, tokio::process::Child), String>`:
     - Create `tokio::process::Command::new(program)`.
     - Set `.args(args).current_dir(cwd)`.
     - On Windows: `#[cfg(windows)] { cmd.creation_flags(CREATE_NEW_PROCESS_GROUP | DETACHED_PROCESS); }`.
     - Set stdout/stderr to `Stdio::piped()` for capture.
     - If `env_vars` provided, set each via `.env(key, value)`. Always inject `AITC_PORT` env var (accept port as parameter or read from a shared AtomicU16).
     - Spawn, extract PID via `child.id().ok_or("no PID")`.
     - Return `Ok((pid, child))` -- caller (the command handler) is responsible for spawning the stdout reader task.
   - `pub async fn terminate_process(pid: u32) -> Result<(), String>`:
     - On Windows: use `tokio::process::Command::new("taskkill").args(&["/PID", &pid.to_string(), "/T"]).output()` for graceful termination. If that fails after 5s timeout, use `/F` flag for force kill.
     - On Unix: use `nix::sys::signal::kill(Pid::from_raw(pid as i32), Signal::SIGTERM)` or fallback to `std::process::Command::new("kill")`.
     - Return Ok(()) on success, Err with message on failure.
   - `pub fn spawn_stdout_reader(mut child: tokio::process::Child, agent_id: String, registry: Arc<AgentRegistry>) -> tokio::task::JoinHandle<()>`:
     - Take stdout from child via `child.stdout.take()`.
     - Spawn tokio task that reads lines from stdout using `tokio::io::BufReader::new(stdout).lines()`.
     - Append each line to the agent's `stdout_buffer` VecDeque in the registry (cap at 1000 lines, pop_front if over).
     - When the child exits (stdout EOF), update agent state to `AgentState::Idle` or `AgentState::Error` based on exit code.
   - Unit tests:
     - `terminate_nonexistent_pid_returns_err`: call terminate_process with PID 999999, expect Err.
     - `launch_detached_echo_returns_pid` (integration-ish): launch `echo hello` (or `cmd /c echo hello` on Windows) as detached, verify PID > 0, wait for exit.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP self-registration endpoint | External processes POST to localhost -- untrusted input |
| Subprocess launch | User-provided cwd and agent type control what binary is spawned |
| Process termination | Agent ID maps to PID that receives kill signal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | self_register.rs | mitigate | Bind to 127.0.0.1 only (no 0.0.0.0). Validate PID in RegisterPayload actually exists as a running process via sysinfo check before accepting registration |
| T-03-05 | Tampering | launch_agent command | mitigate | Validate cwd path exists and is a directory before spawning. Only launch binaries found via PATH lookup (no absolute paths from user input). Reject agent_type values that don't match any registered adapter |
| T-03-06 | Elevation | terminate_agent command | mitigate | Verify the PID being terminated belongs to an agent in the registry (not arbitrary PID kill). Only terminate processes that were either launched by AITC or detected as agents |
| T-03-07 | Denial of Service | self_register endpoint | mitigate | Rate limit: max 10 registrations per second via a simple in-memory counter. Max 100 agents in registry (from Plan 01 T-03-03) |
| T-03-08 | Information Disclosure | get_agent_logs | accept | Stdout logs are local process output on a desktop app. No sensitive data expected beyond what the user already has access to |
</threat_model>
