## Spacing Scale

Inherited from Phase 1 without modification:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status badge padding |
| sm | 8px | Compact element spacing, table cell inner gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, panel margins |
| xl | 32px | Layout gaps between major panels |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions for Phase 3:
- Agent manifest row height: 48px (accommodates agent ID + protocol + status + path in dense layout)
- Conflict alert banner height: 44px (prominent but non-intrusive horizontal strip)
- Status indicator dot: 8px diameter (agent dots in manifest, conflict dots in nav badge)
- System logs panel line-height: 20px (dense log entries per wireframe)

**Copywriting rules (inherited from Phase 1):**
- All UI labels are UPPERCASE with underscores replacing spaces
- Empty state headings use UPPERCASE_UNDERSCORE convention
- Empty state body text uses sentence case for readability
- Numeric values in monospace with leading zeros (e.g., "03" not "3")
- Agent IDs follow wireframe pattern: KAGENT-{4_DIGITS} (e.g., KAGENT-0821)
- File paths displayed in full monospace, truncated with ellipsis from the left if > column width

## Interaction Contracts

### Agent Manifest

| Interaction | Behavior |
|-------------|----------|
| Row hover | surface-container-high background, 150ms transition |
| Row click | Expand row to show agent intent + additional metadata (file list, PID, uptime) |
| Row right-click | Context menu: STOP_AGENT, VIEW_LOGS, COPY_AGENT_ID |
| Status badge click | No action (status is read-only, driven by adapter) |
| Conflict row | Auto-highlighted with error/5 background + 2px error left sliver, no user action needed |

### Deploy Agent Flow

| Step | Behavior |
|------|----------|
| 1. Click DEPLOY_AGENT | Glassmorphism overlay opens with agent type selector |
| 2. Select agent type | Primary left border highlights selected type, working directory input appears |
| 3. Enter working directory | Terminal-style input, auto-suggest from recent directories |
| 4. Optional intent label | Free text input for task description |
| 5. Click LAUNCH_AGENT | Overlay closes, new agent row appears in manifest with RUNNING status, phosphor-in animation |
| 6. Launch failure | Error state inline in the overlay: "LAUNCH_FAILED" + error details, overlay stays open |

### Stop/Terminate Agent Flow

| Step | Behavior |
|------|----------|
| 1. Trigger stop | Via context menu or inline stop icon (Lucide Square icon, 16px) |
| 2. Inline confirmation | Agent row content replaced with confirmation strip (300ms crossfade) |
| 3. Confirm | Agent transitions to stopping state, row removed when process exits |
| 4. Cancel | Confirmation strip dismissed, original row content restored (300ms crossfade) |

### Emergency Halt All

| Step | Behavior |
|------|----------|
| 1. Click EMERGENCY_HALT_ALL | Button background flashes error/20 for 150ms |
| 2. Confirmation | Inline confirmation replaces button: "CONFIRM HALT?" + CONFIRM (error) + CANCEL (ghost) |
| 3. Confirm | All agents receive SIGTERM, manifest clears as processes exit |
| 4. Cancel | Original button restored |

### Conflict Alert Behavior

| Trigger | Response |
|---------|----------|
| New conflict detected | Alert banner slides in at top (phosphor-in), conflict nav badge count increments, agent rows involved get error highlight |
| Dismiss alert | Single alert removed, others shift up. Nav badge count remains until conflict resolves |
| Conflict resolves | Alert auto-dismissed after 5s fade-out, agent rows return to normal status, nav badge count decrements |

### External Agent Detection

| Trigger | Response |
|---------|----------|
| External agent detected | New row appears in manifest with phosphor-in animation, status RUNNING, intent INTENT_UNKNOWN |
| Self-registration POST | Row updates with richer metadata (protocol, intent), status badge may change, 300ms data crossfade |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Tab through manifest rows, Enter to expand, Escape to collapse. Arrow keys within expanded row. |
| Screen reader | Agent status announced on change via aria-live="polite" region. Conflict alerts use aria-live="assertive". |
| Color contrast | All text meets WCAG AA against its background surface. on-surface-variant (#adaaaa) on surface (#0e0e0e) = 9.6:1 ratio. |
| Focus indicators | 2px primary outline on focused interactive elements (ghost border style, visible against dark surfaces) |
| Reduced motion | @media (prefers-reduced-motion: reduce): disable ping animations, replace phosphor-in with instant opacity, remove status transition crossfades |
