Purpose: Addresses D-15 (shared `Arc<tokio::sync::Mutex<ConflictEngine>>`), D-16 (share via axum `Extension` + Tauri `State`). This plan is the pure-mechanical wiring layer — the behavioral change (gate predicate rewrite) lives in Plan 05, which can then edit `hook_handler`'s gate branch without also having to add new infrastructure in the same pass.

Splitting makes each plan verifiable independently: after Plan 04, `cargo test --lib` passes with ZERO behavior change (the engine just happens to be shared now, unused by the hook handler); after Plan 05, the hook handler consumes it and new integration tests light up.
