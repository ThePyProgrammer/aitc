## Accomplishments

- **T1 — Wheel rAF coalescer (D-01..D-04):** `useRafCoalesced` hook + onWheel rewrite. 5 wheel events in a frame now flush as one `setViewport`; summed deltaY preserves "faster scroll → faster zoom"; cursor anchor is last-wins; unmount cancels pending rAF.
- **T2 — Viewport writeback throttle (D-06):** `storeSetViewport(viewport)` now goes through the same coalescer. RadarMinimap + future subscribers re-render at most once per frame even if a future caller bypasses the T1 wheel path.
- **T3 — Folder-hull cache (D-08..D-11):** `getHullCache(nodes, zoom, settledAt)` caches the full per-directory bundle (convex hull → Catmull-Rom spline → Path2D + centroid). `drawFolderHulls` is now a pure draw function; no hull math runs on a settled graph during zoom. Zoom bucketed at 0.1 granularity; settledAt cascade flushes the cache wholesale.
- **T4 — radarPerfDebug diagnostic (D-12, D-13):** `performance.now()` brackets around the render body, gated behind `localStorage.radarPerfDebug === '1'` + a `perfDebugEnabledRef` boolean. Float32Array(120) ring emits `[RadarPerf] p95=… max=… avg=… n=…` once per ~2s window. Zero prod overhead when flag unset.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocker] `vi.spyOn(d3polygon, 'polygonHull')` fails on ESM namespace**

- **Found during:** Task 3 (hullCache unit tests)
- **Issue:** Vitest 3 + ESM yields a read-only module namespace. The plan's test skeleton uses `vi.spyOn(d3polygon, 'polygonHull')` which throws `TypeError: Cannot spy on export "polygonHull". Module namespace is not configurable in ESM.`
- **Fix:** Replaced the `vi.spyOn` pattern with a module-level `vi.mock('d3-polygon', async () => { const actual = await vi.importActual<…>('d3-polygon'); return { ...actual, polygonHull: (...args) => { hullSpy(...args); return actual.polygonHull(...args); }, polygonCentroid: wrapSimilarly }; })`. Module-level `hullSpy`/`centroidSpy` `vi.fn()` references are `.mockClear()`-ed in `beforeEach` and asserted on via `toHaveBeenCalledTimes`.
- **Files modified:** `src/views/Radar/__tests__/hullCache.test.ts`
- **Verification:** 6 Vitest cases pass (D-08 hit, D-09 settledAt change, D-09 null, D-11 bucket change, D-11 bucket stable, bundle shape).
- **Committed in:** `b367489` (Task 3 commit)

**2. [Rule 1 — Bug] T1 draft included an unused `ZOOM_IN_FACTOR` + `void` suppression**

- **Found during:** Task 1 (useCanvasZoomPan rewrite)
- **Issue:** First pass of the rewrite kept the `ZOOM_IN_FACTOR` const alongside the new `Math.pow(ZOOM_OUT_FACTOR, deltaY/100)` exponential factor, guarded with `void ZOOM_IN_FACTOR;` to silence unused-variable warnings. The reciprocal is implicit via `Math.pow` sign behavior, so the const is dead code.
- **Fix:** Removed `ZOOM_IN_FACTOR` const + `void` expression; replaced with an explanatory comment next to `ZOOM_OUT_FACTOR` noting the exponential-form reasoning.
- **Files modified:** `src/hooks/useCanvasZoomPan.ts`
- **Verification:** `npx tsc --noEmit` clean; 4 Vitest cases still pass (D-02 accumulation math confirms identical single-event zoom step at deltaY=-100).
- **Committed in:** `16c663a` (Task 1 commit — fold-in before commit)

**3. [Rule 1 — Bug] T4 negative-guard comment leaked `PerformanceObserver` literal, tripping D-13 grep**

- **Found during:** Task 4 acceptance verification
- **Issue:** The close-bracket comment used "NO PerformanceObserver." as a negative-guard annotation. The D-13 acceptance grep (`! grep -qE "PerformanceObserver"`) does not distinguish comments from code, so the literal identifier tripped the guard.
- **Fix:** Reworded to "No observer API." The intent (D-13 respected) is preserved; the grep now passes.
- **Files modified:** `src/views/Radar/RadarCanvas.tsx`
- **Verification:** `! grep -qE "PerformanceObserver" src/views/Radar/RadarCanvas.tsx` exits 1 (no match) → negative guard passes.
