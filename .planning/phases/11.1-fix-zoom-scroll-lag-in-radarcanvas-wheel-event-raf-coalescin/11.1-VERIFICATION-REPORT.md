## Narrative

Four orthogonal commits; four clean landings. T1 delivers the load-bearing fix: a 44-line generic `useRafCoalesced<T>` hook used in two call sites (wheel + viewport writeback) with `useEffect`-owned cleanup for unmount safety. The wheel handler is now three lines — capture cursor + deltaY in pre-rAF locals, enqueue via merge-fn summing deltaY while last-wins-ing the cursor. The flush runs exactly one `setViewport` per frame regardless of the input event rate — proven by 4 Vitest cases including a 5-events-in-one-frame coalesce test.

T2 (11 net lines, 3 removed) wraps the `storeSetViewport` useEffect with the same coalescer. After T1 lands this is strictly defense-in-depth against future non-wheel callers (keyboard zoom, fit-to-view, resize recomputation), which both the plan and RESEARCH's Pitfall 2 flag honestly. No observable cost; correct scope.

T3 is the architectural extract — `hullCache.ts` owns the expensive convex-hull / Catmull-Rom / Path2D math keyed on a composite `${settledAt}|${zoomBucket}` string, with `_resetHullCacheForTest` exposed for the spec. Zoom bucketing at 0.1 granularity is a documented tradeoff (current padding is `25/zoom`, zoom-dependent; world-space-constant padding would be a visual change and is deferred to 11.2). `drawFolderHulls` is now 47 lines of pure drawing — fill, stroke, arc, fillText — with no polygon math. The dead `polygonHull`/`polygonCentroid`/`smoothHullLine`/`paddedHullPoints` symbols are gone from `GraphRenderer.ts`. 6 Vitest cases cover hit, rebuild-on-epoch-change (settledAt, null settledAt, bucket), bucket-stability, and bundle shape. The SUMMARY's Rule-3 deviation (vi.spyOn is broken on ESM namespaces; use vi.mock with wrappers) is implemented correctly and the pattern is captured in `patterns-established:` for future reuse.

T4 is the diagnostic: four refs, a mount-effect localStorage read with try/catch for private-browsing safety, and 17 lines inside the render loop gated behind `perfDebugEnabledRef.current`. When the flag is off the hot path pays exactly one boolean ref read per frame — no performance.now() call, no ring write, no sort. When the flag is on, `[RadarPerf] p95=... max=... avg=...` emits once per ~2 seconds via Float32Array(120) ring buffer. No `PerformanceObserver` (D-13 respected; comment reworded to "No observer API" per SUMMARY Rule-1 fix).

Carry-over from Phase 11 is respected: the hot-path gate `isSimulatingRef.current` at RadarCanvas.tsx:602 is untouched, the Float32Array read/fallback split at :606-653 is unchanged, and `git diff HEAD~5 -- src/workers/` is zero lines.

The build passes, the type check is clean, and the full test suite shows zero new 11.1-introduced failures against the documented Phase 11 closure baseline. The phase's user-perception acceptance (D-16 manual smoke on a 5k-node Tauri prod build) remains the user's final step — the diagnostic is wired, the code is correct, and everything below the "does it feel fast in the prod build" layer is verified.
