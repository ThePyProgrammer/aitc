- `approve_request(id: i64, pool: State<Pool<Sqlite>>, app_handle: AppHandle) -> Result<(), String>`: UPDATE approval_requests SET status='approved', resolved_at=datetime('now') WHERE id=?. Emit "approval-resolved" event.
- `deny_request(id: i64, pool: State<Pool<Sqlite>>, app_handle: AppHandle) -> Result<(), String>`: UPDATE status='denied', resolved_at. Emit "approval-resolved" event.

- `approve_with_edits(id: i64, edited_content: String, pool: State<Pool<Sqlite>>, app_handle: AppHandle) -> Result<(), String>`: UPDATE status='approved', edited_content=edited_content, resolved_at. Emit "approval-resolved" event.

- `send_chat_message(agent_id: String, content: String, pool: State<Pool<Sqlite>>, app_handle: AppHandle) -> Result<ChatMessage, String>`: INSERT into chat_messages with direction='outbound', delivery_status='queued'. Emit "chat-message-sent" event. Return the inserted message.
- `list_chat_messages(agent_id: String, pool: State<Pool<Sqlite>>) -> Result<Vec<ChatMessage>, String>`: SELECT from chat_messages WHERE agent_id=? ORDER BY created_at ASC.
- `update_message_delivery_status(message_id: i64, status: String, pool: State<Pool<Sqlite>>) -> Result<(), String>`: UPDATE chat_messages SET delivery_status=? WHERE id=?. Validates status is one of 'delivered', 'queued', 'unsupported'.

- `remove_protected_path(id: i64, pool: State<Pool<Sqlite>>) -> Result<(), String>`: DELETE FROM protected_paths WHERE id=?.

All SQL queries MUST use parameterized queries (`sqlx::query!` or `sqlx::query("...").bind(val)`) for T-04-01 input validation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> Rust commands | All approval/chat mutations come from frontend invoke() calls |
| Agent -> Approval system | Agents create approval requests via adapter hooks or protected path triggers |
| DB -> Frontend | Query results returned to frontend for rendering |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | comms/commands.rs (SQL queries) | mitigate | Use sqlx parameterized queries for all DB operations -- never interpolate user input into SQL strings |
| T-04-02 | Tampering | comms/commands.rs (chat messages) | mitigate | Chat message content stored as plain text, rendered as text nodes in React (default JSX escaping). Never use dangerouslySetInnerHTML |
| T-04-03 | Spoofing | comms/commands.rs (create_approval_request) | mitigate | Approval requests created only by Rust backend (adapter hooks or protected path detection). Frontend can only approve/deny/ask, not create fake requests |
| T-04-04 | Denial of Service | protected_paths glob patterns | accept | Glob patterns stored as text, validated at use time. Complexity is bounded by single-file matching scope. Low risk for desktop app |
| T-04-05 | Information Disclosure | diff_content in approval_requests | accept | Diff content is local file data on user's own machine. No cross-user exposure risk |
</threat_model>
