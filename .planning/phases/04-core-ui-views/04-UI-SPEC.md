# Phase 4 -- UI Design Contract

> Visual and interaction contract for the Core UI Views phase (Communications Hub, Airspace Radar, OS Notifications). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Command Horizon design system) |
| Preset | not applicable |
| Component library | none (hand-built to Command Horizon spec) |
| Icon library | Lucide React (stroke-width: 1.5px globally) |
| Fonts | Space Grotesk (headlines, labels, nav), JetBrains Mono (data, code, diffs, chat, body) |

**Source:** Inherits Phase 1 design system. CLAUDE.md explicitly rejects shadcn/ui. All components hand-built to Command Horizon specification from wireframes/vector_terminal/DESIGN.md.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status badge padding, diff line number gutter |
| sm | 8px | Compact element spacing, request queue item padding, chat message gap |
| md | 16px | Default element spacing, panel internal padding, approval card padding |
| lg | 24px | Section padding, panel group spacing, radar manifest panel padding |
| xl | 32px | Layout gaps between 3 Comms Hub panels |
| 2xl | 48px | Panel header area height |
| 3xl | 64px | Page-level vertical spacing (unused in Phase 4) |

Exceptions:
- Comms Hub left panel (request queue): 280px fixed width (from wireframe)
- Comms Hub right panel (telemetry + chats): 260px fixed width (from wireframe)
- Comms Hub center panel: fills remaining space (flex-1)
- Radar right-side agent manifest panel: 280px width, collapsible to 0px
- Mini chat card height: 120px collapsed, auto expanded
- Approval action button row height: 44px (touch target minimum)
- Diff line height: 20px (monospace line spacing for code readability)
- Chat input height: 40px minimum, auto-grows to 120px max

## Typography

| Role | Font | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|------|--------|-------------|----------------|-------|
| Display | Space Grotesk | 20px | 700 (bold) | 1.2 | -0.025em | View titles: "COMMUNICATIONS_HUB", "AIRSPACE_RADAR" |
| Heading | Space Grotesk | 14px | 700 (bold) | 1.4 | 0.1em | Section headers: "PENDING_APPROVALS", "AGENT_MANIFEST", "SYSTEM_LOAD", all UPPERCASE |
| Label | Space Grotesk | 10px | 400 (regular) | 1.4 | 0.1em | Metadata labels: "TARGET:", "STATUS:", "URGENCY:", timestamps, all UPPERCASE |
| Data | JetBrains Mono | 12px | 700 (bold) | 1.4 | -0.025em | Agent IDs, file paths in request cards, radar directory labels |
| Data-sm | JetBrains Mono | 10px | 400 (regular) | 1.4 | -0.025em | Timestamps, delivery status indicators, radar file labels at zoom 3x+ |
| Body | JetBrains Mono | 14px | 400 (regular) | 1.5 | normal | Chat message content, request descriptions, approval detail text |
| Code | JetBrains Mono | 13px | 400 (regular) | 1.5 | normal | Diff code lines (inline diff view), editable diff lines |

**Rules (inherited from Phase 1 + Phase 4 additions):**
- Space Grotesk is always UPPERCASE with wide tracking for labels and nav
- JetBrains Mono is used for all data display, file paths, code diffs, and chat messages
- Never use pure white (#FFFFFF) for body text; use on-surface-variant (#adaaaa)
- Reserve on-surface (#ffffff) for critical headers, active elements, and agent names in manifest
- Diff added lines: Code role in primary (#8eff71) on primary/5 background
- Diff removed lines: Code role in error (#ff7351) on error/5 background with strikethrough
- Diff unchanged lines: Code role in on-surface-variant (#adaaaa)

### The No-Line Rule (Inherited)

Borders prohibited for sectioning. Phase 4 boundaries defined through:
1. Tonal shifts between the 3 Comms Hub panels (surface-container-low | surface-container-highest | surface-container)
2. Negative space -- 4px gap between panels
3. Ghost borders (outline-variant at 10% opacity) permitted only for: diff code block container, chat message thread boundary, radar manifest panel edge

## Interaction Contracts

### Communications Hub -- Request Queue

- Click request card: selects it, loads detail in center panel, highlights with surface-container-high bg + 2px primary left border
- Scroll: virtualized list via TanStack Virtual, overscan 5 items
- New request arrives: card animates in at top with phosphor transition, PendingCountBadge on nav pulses
- Empty state: show "NO_PENDING_REQUESTS" with blinking cursor animation

### Communications Hub -- Approval Workflow

- "APPROVE" click: request status changes to approved, card moves to resolved section or removes from queue, OS notification sent to agent
- "DENY" click: request status changes to denied, card removes from queue
- "ASK_FOR_MORE_INFO" click: reveals inline text input below the button row, user types question, press Enter or click Send to submit, request stays in pending state with "INFO_REQUESTED" sub-status
- "APPROVE_WITH_EDITS": user clicks an added line in the diff to make it contentEditable, edits inline, "APPROVE_WITH_EDITS" button appears replacing "APPROVE", click to submit with modified content
- Edit mode: diff content frozen (incoming updates suppressed for this request), stale indicator shown if updates arrived while editing

### Communications Hub -- Chat

- Type message in ChatInput, press Enter or click Send icon
- Message appears immediately in thread with "queued" delivery status (optimistic insertion)
- Delivery status updates to "delivered" or "unsupported" when backend confirms
- Messages sorted by createdAt timestamp
- Thread scrolls to bottom on new message
- Mini chat card in right panel shows last message preview, unread count badge (primary bg)
- Click mini chat card agent name: center panel switches to that agent's full chat thread

### Airspace Radar -- Navigation

- Mouse wheel: zoom in/out (0.5x to 20x), zoom toward cursor position
- Click + drag: pan the viewport
- Progressive detail rendering:
  - Zoom 1x: directory rectangles + directory labels (Space Grotesk 10px UPPERCASE) + agent dots with pulse
  - Zoom 3x+: file-level rectangles + file labels (JetBrains Mono 10px) + lead lines with timestamps
  - Zoom 8x+: full file details + complete trajectory history
- Sub-pixel rectangles culled (width or height < 1px at current zoom)

### Airspace Radar -- Agent Interaction

- Hover agent dot: show AgentTooltip (HTML overlay, glassmorphism) with agent ID, status, file count, intent
- Click agent dot: select agent, highlight on radar (ambient glow), scroll manifest panel to agent, show AlertDetail
- Click agent in manifest panel: center radar viewport on agent dot, select agent
- Manifest panel collapse toggle: slide panel to 0px width, radar canvas expands to fill

### Airspace Radar -- Minimap

- Shows full treemap at minimum zoom in 160x120px overlay
- White rectangle indicates current viewport bounds
- Click minimap: jump viewport to clicked position

### Native OS Notifications

- New pending approval request: fire native OS notification with title "APPROVAL_REQUIRED", body includes agent ID and target file path
- Conflict detected (from Phase 3): fire notification with title "CONFLICT_DETECTED", body includes conflicting agents and file
- Click notification: bring app window to foreground, navigate to relevant view (Comms for approvals, Conflicts for conflicts)
- System tray icon updates: green (healthy), amber (pending approvals), red (active conflicts)
- Notifications respect Phase 3's per-state configurable settings (D-09 from Phase 3 CONTEXT)

### Keyboard Shortcuts

- Ctrl+K / Cmd+K: command palette (inherited from Phase 1)
- Within Comms Hub:
  - Arrow Up/Down: navigate request queue
  - Enter: select highlighted request
  - A: approve selected request
  - D: deny selected request
  - Escape: deselect / exit edit mode

| Element | Copy |
|---------|------|
| Comms Hub view title | COMMUNICATIONS_HUB |
| Radar view title | AIRSPACE_RADAR |
| Pending approvals header | PENDING_APPROVALS |
| Active requests badge | {N}_ACTIVE_REQUESTS |
| Agent manifest header | AGENT_MANIFEST |
| System load header | SYSTEM_LOAD |
| Telemetry feed header | TELEMETRY_FEED |
| CPU label | CPU_CLUSTER |
| Memory label | MEMORY_SNAP |
| Primary CTA (approve) | APPROVE |
| Deny CTA | DENY |
| Ask more info CTA | ASK_FOR_MORE_INFO |
| Approve with edits CTA | APPROVE_WITH_EDITS |
| Chat input placeholder | TYPE_COMMAND_OR_QUERY... |
| Delivery status: delivered | DELIVERED |
| Delivery status: queued | QUEUED |
| Delivery status: unsupported | UNSUPPORTED |
| Comms empty state heading | NO_PENDING_REQUESTS |
| Comms empty state body | All channels clear. Approval requests from agents will appear here when action is needed. |
| Radar empty state heading | AWAITING_SIGNAL |
| Radar empty state body | No active watch session. Start a file watcher to populate the airspace map. |
| Chat empty state heading | NO_MESSAGES |
| Chat empty state body | Send a message to begin communication with this agent. |
| Notification: approval required | APPROVAL_REQUIRED -- {agent_id} requests access to {file_path} |
| Notification: conflict detected | CONFLICT_DETECTED -- {agent_a} and {agent_b} both writing to {file_path} |
| Error: backend disconnected | SIGNAL_LOST -- Connection to backend interrupted. Check system tray for status. |
| Error: agent unreachable | AGENT_UNREACHABLE -- Unable to deliver message to {agent_id}. Agent may have terminated. |
| Stale data indicator | DATA_STALE -- Updates paused during edit mode. Changes will sync on save. |
| Radar no agents | AIRSPACE_CLEAR -- No agents detected. Deploy or attach agents to populate the radar. |

**Copywriting rules (inherited from Phase 1):**
- All UI labels are UPPERCASE with underscores replacing spaces (ATC radio aesthetic)
- Empty state headings use the same UPPERCASE_UNDERSCORE convention
- Empty state body text uses sentence case for readability
- Numeric values displayed in monospace with leading zeros where applicable (e.g., "03" not "3")
- Agent IDs displayed verbatim as received from the adapter (no case transformation)
- File paths displayed relative to watched root, truncated with ellipsis if > 40 chars
