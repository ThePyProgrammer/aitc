**Create `src/stores/radarStore.ts`:**

```typescript
import { create } from 'zustand';
import { invoke } from '@tauri-apps/api/core';

export interface TreeIndexEntry {
  path: string;
  size: number;
  isDir: boolean;
  depth: number;
}

export interface Viewport {
  zoom: number;
  panX: number;
  panY: number;
}

// 8-color agent dot palette per UI-SPEC
export const AGENT_DOT_PALETTE = [
  '#8eff71', '#00cffc', '#ffd16f', '#ff7351',
  '#c084fc', '#f472b6', '#67e8f9', '#a3e635',
];

// Hash agent ID to palette index for session-consistent colors
export function getAgentColor(agentId: string): string {
  let hash = 0;
  for (let i = 0; i < agentId.length; i++) {
    hash = ((hash << 5) - hash + agentId.charCodeAt(i)) | 0;
  }
  return AGENT_DOT_PALETTE[Math.abs(hash) % AGENT_DOT_PALETTE.length];
}

interface RadarStore {
  treeData: TreeIndexEntry[];
  viewport: Viewport;
  selectedAgentId: string | null;
  isManifestOpen: boolean;
  fetchTreeIndex: () => Promise<void>;
  setViewport: (v: Partial<Viewport>) => void;
  selectAgent: (id: string | null) => void;
  toggleManifest: () => void;
  reset: () => void;
}
```

Implement: `fetchTreeIndex` calls `invoke<TreeIndexEntry[]>('get_tree_index')` and sets `treeData`. `setViewport` merges partial viewport. `selectAgent` sets selectedAgentId. `toggleManifest` flips `isManifestOpen`. Default viewport: `{ zoom: 1, panX: 0, panY: 0 }`. Default `isManifestOpen: true`.

**Create `src/hooks/useTreemapLayout.ts`:**

```typescript
import { useMemo } from 'react';
import squarify from 'squarify';
import type { TreeIndexEntry } from '../stores/radarStore';

export interface FileTreeNode {
  path: string;
  name: string;
  size: number;
  isDir: boolean;
  children: FileTreeNode[];
}

export interface TreemapRect {
  x0: number; y0: number;
  x1: number; y1: number;
  path: string;
  name: string;
  depth: number;
  isFile: boolean;
  size: number;
}
```

Implement `buildFileTree(entries: TreeIndexEntry[]): FileTreeNode` that groups flat file entries by directory segments into a nested tree. Each directory's size is the sum of its children's sizes. Root node represents the repo root.

Implement `computeTreemapLayout(root: FileTreeNode, width: number, height: number): TreemapRect[]` that:
1. For each level of the tree, uses `squarify(children.map(c => ({ ...c, value: c.size || 1 })), container)` to compute rectangle positions
2. Recursively computes layouts for sub-directories within their parent rectangle bounds
3. Returns a flat array of all TreemapRect objects with depth, path, name, isFile

Export `useTreemapLayout(treeData: TreeIndexEntry[], width: number, height: number): TreemapRect[]` hook that:
- Calls `buildFileTree` then `computeTreemapLayout`
- Memoizes with `useMemo` keyed on `[treeData, width, height]` to prevent recomputation on every render (Pitfall 2 from research)

- `viewport` state: `{ zoom: number, panX: number, panY: number }`
- `onWheel(e: WheelEvent)`: zoom toward cursor, factor 0.9/1.1 per scroll delta, clamped to [0.5, 20]

- `onMouseMove(e: MouseEvent)`: if dragging, update panX/panY by delta

- `screenToWorld(screenX, screenY)`: convert screen coords to world coords: `worldX = (screenX - panX) / zoom`
- Return `{ viewport, setViewport, handlers: { onWheel, onMouseDown, onMouseMove, onMouseUp }, screenToWorld }`

**Create `src/views/Radar/RadarCanvas.tsx`:**

Core Canvas 2D renderer implementing D-09, D-10, D-11 and UI-SPEC RadarCanvas/TreemapCell/AgentDot specs:

1. **Canvas setup:**
   - `<canvas>` element fills parent container (use `ResizeObserver` to track container size)
   - HiDPI scaling per Pitfall 4: set canvas width/height to `Math.floor(cssWidth * devicePixelRatio)` x `Math.floor(cssHeight * devicePixelRatio)`, CSS width/height to logical size, `ctx.scale(devicePixelRatio, devicePixelRatio)`

2. **Treemap rendering via useTreemapLayout:**
   - Get `treeData` from `useRadarStore`
   - Compute layout with `useTreemapLayout(treeData, canvasWidth, canvasHeight)`
   - Store layout in a ref (`layoutRef`) to avoid recomputation on viewport changes

3. **Render loop (requestAnimationFrame + dirty tracking per Pattern 2):**
   - `dirtyRef` boolean flag, set true when viewport, agents, or layout change
   - On each frame: if dirty, clear canvas, apply viewport transform via `ctx.setTransform(zoom, 0, 0, zoom, panX, panY)`, draw treemap, draw agent dots, set dirty=false
   - Cancel animation frame on unmount

4. **drawTreemap function (progressive detail per D-11, Pattern 3):**
   - For each TreemapRect in layout:
     - Skip if screen-space width or height < 1px (sub-pixel culling per VIZN-04)
     - Directory rectangles: stroke outline-variant (#494847) at 30% opacity, fill surface-container-low (#131313)
     - File cells: stroke outline-variant at 15% opacity, fill surface (#0e0e0e)
     - At zoom >= 1: draw directory labels (Space Grotesk 10px UPPERCASE) if screen-space width > 60px
     - At zoom >= 3: draw file labels (JetBrains Mono 10px) if screen-space width > 40px
     - At zoom >= 8 and isFile: draw file details (size)

5. **drawAgentDots function (D-10):**
   - Get agents from `useAgentStore`
   - Get recent file events from `usePipelineStore` to determine which file each agent is touching
   - For each agent, find the TreemapRect matching the agent's most recent file event path
   - Draw 8px diameter circle at center of that rect, filled with `getAgentColor(agent.id)`
   - Draw pulse animation: 2 concentric rings at 30% and 20% opacity, scale 1 to 2.5 over 2s cycle using `(Date.now() % 2000) / 2000` for animation phase
   - When pulse is active, set dirty=true each frame to keep animating

6. **Zoom/Pan wiring:**
   - Attach `useCanvasZoomPan` handlers to canvas element
   - Sync viewport changes to `radarStore.setViewport()` for persistence
   - Viewport from useCanvasZoomPan drives the ctx.setTransform in render loop

7. **Hit testing for agent dot hover:**
   - On mousemove: convert screen coords to world coords via `screenToWorld`
   - Check if world coords are within 8px radius of any agent dot position
   - If hovering an agent dot: set `hoveredAgentId` in local state (tooltip rendered in Plan 05)
