<objective>
Build the PID attribution layer (FMON-02): a `ProcessSnapshot` that polls `sysinfo` for processes matching a name allowlist (claude, claude-code, codex, opencode per D-07), caches their cwd/exe/cmd/parent, and exposes an `attribute(path)` function that maps an absolute filesystem path to a PID by prefix-matching candidate cwds. The attribution is best-effort per D-06 — exact attribution requires ETW which is deferred to a future phase. This plan also provides an `AttributingStream` wrapper that rewrites the `Attribution` field of FileEvents produced by Plan 02's watcher in-flight.

Purpose: The research flagged four concerns this plan addresses empirically: (1) sysinfo refresh cost on Windows (Plan 01's Wave 0 benchmark gives the measured number — this plan consumes it to choose polling cadence), (2) Windows PEB read may return `None` for `cwd` on some processes without elevation — attribution must handle None gracefully, (3) process name allowlist needs to be documented and easy to extend when Phase 3 adds new agent types, (4) the attribution logic must not run per-event (too expensive) — snapshot ticks independently on a 1000ms interval.

Output: `ProcessSnapshot` is a self-contained struct (no watcher coupling) so it can be tested in isolation with `std::process::Command`-spawned test fixtures. `AttributingStream` is the composition point: Plan 04 plumbs Plan 02's watcher receiver through `AttributingStream` before forwarding to the Tauri Channel.
</objective>

//! Process snapshot + PID attribution heuristic (FMON-02, D-05/D-06/D-07).
//!
//! Per Claude's Discretion D-07: discover by process NAME allowlist + cwd
//! prefix match. Watched-directory-only attribution would need OS file-handle
//! inspection (expensive on Windows).
//!
//! Per D-06: best-effort in Phase 2. Phase 3 agent adapters will self-report
//! (pid, path) claims that supplement this heuristic for higher accuracy.
//!
//! Research note (02-RESEARCH.md Pitfall 4): sysinfo::refresh_processes on
//! Windows costs 30-100ms due to PEB reads for cwd. We use ProcessRefreshKind
//! to narrow the refresh and cache the result behind a 1000ms tick.

use crate::pipeline::events::{Attribution, FileEvent, FileEventBatch};
use serde::{Deserialize, Serialize};
use specta::Type;
use std::collections::HashMap;
use std::path::{Path, PathBuf};
use std::sync::Arc;
use std::time::Duration;
use sysinfo::{Pid, ProcessRefreshKind, ProcessesToUpdate, System, UpdateKind};
use tokio::sync::{mpsc, RwLock};

/// Agent process names we attempt to attribute to.
/// Matching is lowercased + substring, so `claude.exe`, `Claude Code.exe`,
/// `codex-cli` all match.
///
/// Extending: Phase 3 agent adapters should grow this list (or switch to a
/// runtime-configurable allowlist) when they add new agent types.
pub const AGENT_NAME_ALLOWLIST: &[&str] = &["claude", "claude-code", "codex", "opencode"];

/// Frontend-facing process info (serializable via specta).
#[derive(Debug, Clone, Serialize, Deserialize, Type)]
#[serde(rename_all = "camelCase")]
pub struct ProcessInfo {
    pub pid: u32,
    pub name: String,
    pub cwd: Option<PathBuf>,
    pub exe: Option<PathBuf>,
    pub parent_pid: Option<u32>,
}

/// Internal candidate kept in the snapshot. We deliberately clone into
/// ProcessInfo when exposing to the frontend to avoid borrowing sysinfo types
/// across the specta boundary.
#[derive(Debug, Clone)]
struct CandidateProc {
    pid: u32,
    name: String,
    cwd: PathBuf, // non-optional: only candidates with a known cwd make it here
    exe: Option<PathBuf>,
    parent: Option<u32>,
}

pub struct ProcessSnapshot {
    sys: System,
    candidates: HashMap<u32, CandidateProc>,
    allowlist: Vec<&'static str>,
}

impl ProcessSnapshot {
    pub fn new() -> Self {
        Self {
            sys: System::new(),
            candidates: HashMap::new(),
            allowlist: AGENT_NAME_ALLOWLIST.to_vec(),
        }
    }

    /// Create a snapshot with a custom allowlist. Intended for tests that need
    /// to include the current test-runner binary name.
    pub fn with_allowlist(allowlist: Vec<&'static str>) -> Self {
        Self {
            sys: System::new(),
            candidates: HashMap::new(),
            allowlist,
        }
    }

    /// Refresh process table, narrow to allowlisted candidates, cache results.
    pub fn refresh(&mut self) {
        // Narrow refresh to only what we need. cwd is always updated (agents can
        // change dir mid-session on Windows; also PEB reads are cheapest when
        // scoped). cmd and exe are only loaded once per process.
        self.sys.refresh_processes_specifics(
            ProcessesToUpdate::All,
            true, // remove dead processes from the table
            ProcessRefreshKind::nothing()
                .with_cwd(UpdateKind::Always)
                .with_cmd(UpdateKind::OnlyIfNotSet)
                .with_exe(UpdateKind::OnlyIfNotSet),
        );

        self.candidates.clear();
        for (pid, proc) in self.sys.processes() {
            let raw_name = proc.name().to_string_lossy().to_lowercase();
            let is_agent = self.allowlist.iter().any(|a| raw_name.contains(a));
            if !is_agent {
                continue;
            }
            // Windows: cwd() can return None if PEB read fails (insufficient perms
            // for cross-user processes, or sysinfo falls back to PROCESS_QUERY_LIMITED_INFORMATION).
            // Per D-06, skip those silently — they count as "unattributed" for any event.
            let Some(cwd_path) = proc.cwd() else {
                continue;
            };
            self.candidates.insert(
                pid.as_u32(),
                CandidateProc {
                    pid: pid.as_u32(),
                    name: raw_name,
                    cwd: cwd_path.to_path_buf(),
                    exe: proc.exe().map(|p| p.to_path_buf()),
                    parent: proc.parent().map(|p| p.as_u32()),
                },
            );
        }
    }

    /// Frontend-facing candidate list.
    pub fn candidates(&self) -> Vec<ProcessInfo> {
        self.candidates
            .values()
            .map(|c| ProcessInfo {
                pid: c.pid,
                name: c.name.clone(),
                cwd: Some(c.cwd.clone()),
                exe: c.exe.clone(),
                parent_pid: c.parent,
            })
            .collect()
    }

    /// Attribution heuristic per 02-RESEARCH.md Pattern 3:
    ///   - 0 candidates whose cwd is a prefix of the event path → Unattributed
    ///   - 1 candidate                                            → Pid
    ///   - ≥2 candidates                                          → Ambiguous
    pub fn attribute(&self, event_path: &Path) -> Attribution {
        let matches: Vec<u32> = self
            .candidates
            .values()
            .filter(|c| event_path.starts_with(&c.cwd))
            .map(|c| c.pid)
            .collect();
        match matches.len() {
            0 => Attribution::Unattributed,
            1 => Attribution::Pid(matches[0]),
            _ => Attribution::Ambiguous(matches),
        }
    }
}

impl Default for ProcessSnapshot {
    fn default() -> Self {
        Self::new()
    }
}

/// Spawn a tokio task that refreshes `snapshot` every `interval`. Returns the
/// JoinHandle; dropping it (without aborting) lets the task run for the app
/// lifetime. Plan 04's WatcherHandle should hold this JoinHandle and abort it
/// on stop_watch.
pub fn spawn_snapshot_refresher(
    snapshot: Arc<RwLock<ProcessSnapshot>>,
    interval: Duration,
) -> tokio::task::JoinHandle<()> {
    tokio::spawn(async move {
        let mut tick = tokio::time::interval(interval);
        loop {
            tick.tick().await;
            // Write-lock only during the refresh; release before the next tick.
            let mut snap = snapshot.write().await;
            snap.refresh();
        }
    })
}

/// Spawn a tokio task that consumes FileEventBatch values from `in_rx`,
/// rewrites each event's `attribution` field based on the current snapshot,
/// and forwards the batch to `out_tx`. The snapshot is read-locked per batch
/// (not per event) to minimize contention.
pub fn start_attributing_stream(
    mut in_rx: mpsc::Receiver<FileEventBatch>,
    out_tx: mpsc::Sender<FileEventBatch>,
    snapshot: Arc<RwLock<ProcessSnapshot>>,
) -> tokio::task::JoinHandle<()> {
    tokio::spawn(async move {
        while let Some(mut batch) = in_rx.recv().await {
            let snap = snapshot.read().await;
            for ev in batch.events.iter_mut() {
                ev.attribution = snap.attribute(&ev.path);
            }
            drop(snap);
            if out_tx.send(batch).await.is_err() {
                break; // consumer gone — exit cleanly
            }
        }
    })
}

        // Use the test binary name as the allowlist. On Windows it's

        snap.refresh();
        // We can't guarantee the test process has a cwd visible to PEB-read,

        let _ = snap.candidates();

|----------|-------------|
| sysinfo → process enumeration | Reading other processes' memory (PEB on Windows); privilege boundary |
| ProcessInfo → IPC boundary | Serialized process info crosses into webview; information disclosure surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | Enumerating and exposing cwd/cmd/exe for other users' processes | mitigate | sysinfo default on Windows uses `PROCESS_QUERY_LIMITED_INFORMATION` which returns `None` for inaccessible processes. Candidates with `cwd() == None` are SKIPPED (not emitted). AGENT_NAME_ALLOWLIST further restricts exposure to only agent-like processes. |
| T-02-03-02 | Information Disclosure | ProcessInfo fields (cwd, exe paths) leaked to webview | mitigate | Only allowlisted processes (agents) are emitted. Agent paths are ones the user already knows. Non-agent user processes (browsers, IDE, etc.) never appear in candidates. |
| T-02-03-03 | Spoofing | Process spoofing — malicious binary renames itself to `claude.exe` to gain attribution | mitigate | Phase 2 best-effort acknowledged (D-06). Phase 3 adapters will verify agents via parent-pid + spawn-path check. Phase 2 doesn't take destructive actions on attributed events; worst case is incorrect UI display which is self-correcting on next agent launch. |
| T-02-03-04 | Denial of Service | sysinfo refresh cost pegging CPU at 1Hz polling | mitigate | ProcessRefreshKind narrowed to cwd+cmd+exe only (no env, no tasks). Plan 01's Wave 0 benchmark proved <500ms per refresh is achievable. Plan 04 will use a 1000ms interval; if Plan 01 benchmark reported ≥50ms, Plan 04 should bump to 2000ms and document in SUMMARY. |
| T-02-03-05 | Tampering | Path prefix match fooled by symlinks pointing inside repo root | accept | Phase 2 uses lexical `event_path.starts_with(candidate.cwd)` without canonicalization. Symlink escape is covered by Plan 02's path_is_under_root check at the watcher boundary. If a symlink points INTO the repo from an agent's cwd, attribution works correctly (the event path is inside the repo, matches the candidate cwd). |
| T-02-03-06 | Tampering | Candidate cwd being a prefix of an unrelated parent directory — wrong PID attributed | accept | The cwd prefix heuristic is best-effort by D-06. For Phase 2 solo-dev workflows the risk is negligible (user controls which agents run). Phase 3 agent adapters supplement with explicit (pid, path) claims. |
| T-02-03-07 | Information Disclosure | ProcessSnapshot candidate JSON emitted via tauri-specta bindings leaks cwd at build time | accept | Build-time bindings expose TYPES only, not values. Runtime values are per-session and never written to disk in Phase 2 (D-04: in-memory only). |
</threat_model>
