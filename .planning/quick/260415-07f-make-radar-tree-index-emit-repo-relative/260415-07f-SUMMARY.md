# Quick Task 260415-07f: Make radar tree_index emit repo-relative paths — Summary

Fix radar treemap rendering absolute OS paths as the visible tree (`C > Users > prann > projects > aitc > src > ...`) by stripping the canonical repo root at the `get_tree_index` IPC boundary; in-memory HashMap keys remain absolute so attribution and reconciliation invariants are preserved.

## What Shipped

- **`ActiveWatch.repo_root: PathBuf`** added to `pipeline_state.rs`, populated in `start_watch` from the existing UNC-stripped `canonical` binding.
- **`serialize_tree_index(tree, repo_root) -> Vec<TreeIndexEntry>`** pure helper in `pipeline/commands.rs` that:
  - `strip_prefix(repo_root)` per entry, skipping entries outside the root (defensive).
  - Filters out the repo-root entry itself (empty relative path) — frontend synthesizes its own root.
  - Replaces `\` with `/` for consistent treemap segment parsing on Windows.
  - Computes depth from the *relative* component count (so `src/foo.rs` → depth 2).
- `get_tree_index` command body reduced to a single call to the helper.
- 5 new unit tests in `serialize_tests` module (no tokio/Tauri needed):
  - `emits_repo_relative_file_path`
  - `emits_repo_relative_dir_path`
  - `skips_repo_root_entry`
  - `forward_slash_normalized`
  - `skips_paths_outside_repo_root`
