**Job A: `check` (matrix)** — runs on `pull_request` and `push` to `main`.
- `strategy.fail-fast: false`
- Matrix: `os: [ubuntu-24.04, windows-latest, macos-latest]`
- Steps:
  1. `actions/checkout@v4`
  2. Install Linux system deps when `matrix.os == 'ubuntu-24.04'`:
     ```
     sudo apt-get update
     sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev libssl-dev libglib2.0-dev libgtk-3-dev patchelf build-essential curl wget file
     ```
  3. `actions/setup-node@v4` with `node-version: 20` and `cache: npm`
  4. `dtolnay/rust-toolchain@stable` (pinned via major) with `components: clippy, rustfmt`
  5. `Swatinem/rust-cache@v2` with `workspaces: src-tauri -> target`
  6. `npm ci`
  7. Frontend: `npm run build` (this runs `tsc` — covers typecheck), then `npm test`
  8. Rust (set `working-directory: src-tauri`):
     - `cargo fmt --check`
     - `cargo clippy --all-targets -- -D warnings` (non-blocking for now: prefix with `continue-on-error: true` ONLY if clippy fails on first run; leave strict by default)
     - `cargo check --all-targets`
     - `cargo test --all-features`

**Job B: `arch-check`** — runs on same triggers, `runs-on: ubuntu-latest`, `container: archlinux:latest`.
- Steps:
  1. Install deps inside container (no sudo; container runs as root):
     ```
     pacman -Syu --noconfirm
     pacman -S --noconfirm --needed base-devel webkit2gtk-4.1 libappindicator-gtk3 librsvg openssl glib2 gtk3 curl wget patchelf nodejs npm rustup git
     rustup default stable
     ```
  2. `actions/checkout@v4`
  3. `Swatinem/rust-cache@v2` with `workspaces: src-tauri -> target` (cache key will include arch runner)
  4. `npm ci`
  5. `npm run build` and `npm test`
  6. `cd src-tauri && cargo check --all-targets && cargo test --all-features`

Use `v4`/`v2` major tags for actions (checkout, setup-node, rust-cache). Use `@stable` for `dtolnay/rust-toolchain`. Do NOT pin to commit SHAs — major-tag pinning is acceptable per constraints.

<task type="auto">
  <name>Task 2: Create release workflow (release.yml) using tauri-action on tag push</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/release.yml` that builds Tauri installers for all three desktop platforms when a `v*` tag is pushed, and uploads them to a draft GitHub Release.

Structure:
```yaml
name: release
on:
  push:
    tags: ['v*']
  workflow_dispatch:

permissions:
  contents: write   # required for tauri-action to create/edit releases

jobs:
  bundle:
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-24.04
            target: ''
          - os: windows-latest
            target: ''
          - os: macos-latest
            target: 'aarch64-apple-darwin'
          - os: macos-latest
            target: 'x86_64-apple-darwin'
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - name: Install Linux deps
        if: matrix.os == 'ubuntu-24.04'
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev libssl-dev libglib2.0-dev libgtk-3-dev patchelf build-essential curl wget file
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.target }}
      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: src-tauri -> target
      - run: npm ci
      - uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          tagName: ${{ github.ref_name }}
          releaseName: 'AITC ${{ github.ref_name }}'
          releaseDraft: true
          prerelease: false
          args: ${{ matrix.target != '' && format('--target {0}', matrix.target) || '' }}
```

Notes:
- `tauri-action@v0` picks up `beforeBuildCommand` from `tauri.conf.json` (runs `npm run build`) — do not duplicate it.
- Draft release so the developer can edit notes before publishing.
- Do NOT run this workflow on every push — tag-only + manual dispatch keeps minutes low.
- Arch is intentionally excluded from release (no standard Arch installer target in tauri-action; AppImage/deb from ubuntu-24.04 is sufficient for Arch users).

Also add/update `.gitignore` to ensure `src-tauri/target/` and `node_modules/` are ignored if not already (non-destructive: read existing `.gitignore` first; only append missing entries).
  </action>
